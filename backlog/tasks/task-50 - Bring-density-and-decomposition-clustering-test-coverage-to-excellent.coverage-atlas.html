how a reference number reaches an assertion

The HDBSCAN fixture pipeline

A scikit-learn run produces a JSON fixture carrying the final labels and probabilities, sklearn's own single-linkage hierarchy, and a tie_free flag with the min_mst_gap it derives from. That payload feeds two assertion paths of different strictness — one bit-exact, one bounded — and the tie_free flag decides which strictness the full-estimator path is held to.

why most assertions are now exact

Distinct MST weights make the clustering unique

The generator computes the minimum gap between sorted single-linkage merge distances. When every weight is distinct, the minimum spanning tree is mathematically unique, which makes the whole flat clustering unique — so the implementation must reproduce sklearn's labels (up to permutation) and probabilities to 1e-6. A fixture is tie_free exactly when that gap is positive; 12 of the 33 HDBSCAN fixtures are tie-free.

Making the exact tier honest required one production change: HDBSCAN's native-metric distances move from the float32 TensorFlow.js path to plain-JS float64. The rest of the pipeline (MST, condensed tree) is already float64, and the float32 round-trip was the only thing perturbing mutual-reachability weights enough to break bit-level agreement. After the move, most previously tolerance-only fixtures match sklearn exactly.

the measured bound

Across the 21 tie-bound fixtures, the documented probability bound is MAE ≤ 0.16. The observed worst case is 0.150 on the heavily-tied precomputed-cosine fixture; every other fixture sits at or below 0.077. The condensed-tree core (Path A) stays bit-exact for all 33 fixtures because it consumes sklearn's hierarchy directly and never resolves a tie itself.

single dense cluster, resolved

With allow_single_cluster=False — the only mode the estimator supports — HDBSCAN structurally cannot emit exactly one cluster: candidate clusters are born in sibling pairs during condensation, so a lone dense blob is root-only and returns all-noise. The single_blob fixture pins sklearn's actual output (every label -1) rather than a wished-for single label.

KMeans cosine path

Spherical k-means is validated against a scikit-learn reference fixture in generate_kmeans.py — normalize(X) then euclidean KMeans — stored as __fixtures__/kmeans/cosine_directions_n3.json. The colocated parity test asserts centroids and labels up to permutation.

how strict each fixture is held

The probability assertion tiers

Each fixture is classified by its tie_free flag, then asserted as strictly as that classification allows. Tie-free fixtures assert probabilities exactly to 1e-6; tie-bound fixtures carry the documented MAE ≤ 0.16, with the count of affected boundary points stated.

library-defined, no sklearn equivalent

Exemplars and medoids: behavioural coverage

exemplar_indices_ (HDBSCAN, the most-persistent point per cluster) and medoid_indices_ (the real sample nearest each cluster mean) have no scikit-learn counterpart to diff against. They are documented in code and docs/API.md as library-defined and pinned by behaviour: correct count, valid in-range indices, deterministic tie-break.

how regressions fail the build

The coverage gate, single-sourced

jest.coverage.config.js holds GATED_MODULES as the single source of truth. The same array drives instrumentation (collectCoverageFrom), the per-file thresholds (branch ≥ 90, statements / lines / functions ≥ 95), and the suite selection (testMatch derived by colocation convention). It runs as npm run test:coverage:gate on one CI cell (ubuntu / Node 22, ~16 s).

jest.coverage.config.jssingle source of truth
const GATED_MODULES = [
  'src/clustering/hdbscan.ts',
  'src/clustering/medoid_selection.ts',
  'src/decomposition/pca.ts',
  'src/distance/kdistance.ts',
  'src/graph/condensation_tree.ts',
  'src/graph/minimum_spanning_tree.ts',
  'src/graph/mutual_reachability.ts',
  // representations.ts is type-only — covered behaviourally, no threshold
];
const PER_FILE_THRESHOLD = { branches: 90, statements: 95, lines: 95, functions: 95 };

module.exports = {
  collectCoverageFrom: GATED_MODULES,                       // instrumentation
  coverageThreshold: Object.fromEntries(                   // per-file gate
    GATED_MODULES.map((file) => [file, PER_FILE_THRESHOLD])),
  testMatch: GATED_MODULES.map(/* colocated *.test.ts */), // suite selection
};
deliberate sklearn deviations, pinned in docs

docs/API.md records the two intentional differences from scikit-learn, both pinned by tests: HDBSCAN returns all-noise for a single sample, and clamps min_samples to n — sklearn raises in both cases. The review round also removed the vestigial codecov upload step, which had referenced a never-generated lcov file since its introduction.
