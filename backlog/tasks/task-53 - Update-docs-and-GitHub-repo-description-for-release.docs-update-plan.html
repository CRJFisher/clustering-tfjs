HDBSCAN API — as committed

new HDBSCAN({ min_cluster_size?: number; // integer >= 2, default 5 min_samples?: number; // integer >= 1, default = min_cluster_size metric?: 'euclidean' | 'manhattan' | 'precomputed'; // default 'euclidean' cluster_selection_method?: 'eom' | 'leaf'; // default 'eom' cluster_selection_epsilon?: number; // >= 0, default 0 })
min_cluster_size
number · int ≥ 2
Smallest grouping treated as a cluster; smaller groupings become noise.
default: 5
min_samples
number · int ≥ 1
How conservative the density estimate is — higher means more points labelled noise.
default: = min_cluster_size
metric
'euclidean' | 'manhattan' | 'precomputed'
Distance measure; 'precomputed' takes a square, symmetric, zero-diagonal distance matrix directly.
default: 'euclidean'
+ 'manhattan' added in review
cluster_selection_method
'eom' | 'leaf'
How clusters are extracted from the hierarchy — Excess of Mass vs. leaf clusters.
default: 'eom'
cluster_selection_epsilon
number · ≥ 0
Distance threshold below which clusters are merged; 0 disables the merge.
default: 0
+ added in review
Post-fit attributes — added in review

After fit, labels_ holds per-point cluster assignments (-1 for noise) and probabilities_ holds per-point membership strength (0 for noise). Because HDBSCAN determines its own cluster count, it is not a valid algorithm for find_optimal_clusters — the post-fit prose states this explicitly.
