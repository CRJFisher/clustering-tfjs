An optimistic navigator.gpu probe warns before the race; an authoritative actual_backend reconciliation after it is the source of truth — so the page never claims a WebGPU win it did not measure, and never shows an empty lane.

Comprehension job 1 · the crux · two phases, one source of truth

Toggle whether this browser has WebGPU, then run the race — watch which states the page enters

Why two checks instead of one

The main-thread navigator.gpu probe is only a signal — a browser can expose navigator.gpu yet still fail to initialize the backend. So the worker's post-init tf.getBackend() is returned as actual_backend and reconcile_fallback trusts only that: a "webgpu" result clears the banner, anything else names the real backend and reveals the payoff clip.

Comprehension job 2 · the chain · why a GPU lane is never broken or blank

The GPU lane walks webgpu → webgl → wasm → cpu until one backend initializes

The honest-label contract (race_protocol.ts)

A lane is requested as "webgpu" but the result carries actual_backend = tf.getBackend() — whatever truly ran. The UI relabels from that: a fell-back lane reports WEBGL in its panel header, so the page can never advertise a backend it did not measure. The same result now also carries tfjs_version = tf.version_core.

Comprehension job 3 · methodology-as-a-feature · the config can't drift from the code

The "how this is timed" panel reads live values from the real result — so it can't lie

The methodology expander is always available — not gated on fallback

Unlike the banner and the reference clip (which appear only to non-WebGPU visitors), the <details> expander and the "YOUR browser/GPU" footer are always present, on every visit. Surviving scrutiny means the timing protocol is one click away whether or not a fallback happened.
