Comprehension job 2 · the wiring · run_race callbacks → live UI

The harness callbacks drive every pixel — and rendering stays out of the clock

The one hard rule

Scatter rendering is a setup step, not a per-rep step: the 2-D projection and both canvas draws happen before run_race is awaited.

The 2-D projection and both canvas draws happen in on_dataset, before the race. The live timer animates from on_progress pings — an rAF stopwatch reporting elapsed wall-clock only, never the median.

Comprehension job 3 · judgment call A · resolved with rationale

The dataset is 32-D; the scatter is 2-D — project once with a tiny deterministic PCA

Why this is the honest choice

PCA is deterministic from the fixed seed — every visitor sees the same picture, reproducible for the later permalink.

It is pure JS, so the main thread never initializes a tfjs engine — all compute stays in the workers (per make_blobs_js.ts).

Coloring by ground-truth label, not either backend's predictions, keeps the scatter a fixed reference image of the structure both lanes cluster — never a per-run output.

Comprehension job 4 · judgment call B · resolved with rationale

The "same result" check runs on the affinity checksum — not on fit_predict labels

What the indicator shows — three states, exact copy

Match (symmetric relative gap ≤ 1e-3): "Same result — both backends agree on the RBF affinity matrix (matching checksum)." It says matching checksum, deliberately not "identical matrix" — the checksum is a scalar sum of the affinity entries, not a bit-identical comparison of all ~4M cells.

Diverged (gap exceeds the tolerance): "Results diverged — one backend fell back or computed a different kernel."

Error (a lane crashed or timed out): a distinct neutral "Race incomplete" state — never the divergence warning, because no comparison was produced. The badge is backed by a real computed equality on the timed output, never a hard-coded checkmark.
