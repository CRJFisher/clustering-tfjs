Why Spectral affinity, not KMeans

RBF affinity is one big matmul/exp with a single readback before the eigensolver — the cleanest, largest, most defensible gap (~21–26×).

KMeans is the trap: its Lloyd loop reads back every iteration and reduces centroids in a pure-JS double loop, stalling the GPU on the bus for only ~1.2×.

So KMeans appears in the grid as convergence eye-candy, never as the headline multiplier.

Comprehension job 5 · main thread orchestrates, workers compute

One seeded dataset, posted identically to both lanes

Robust lane lifecycle

Lanes run via Promise.allSettled with a per-lane timeout — a hung or failed lane never strands its sibling worker or freezes the UI, and each lane terminates its own worker on settle.

Inside the worker the input tensor is disposed in a finally, so even a mid-run backend error releases it cleanly.

Honest headline

The orchestrator surfaces the GPU lane's actual tf.getBackend(), and the "N× faster" headline names that real backend — a WebGPU→WebGL fallback is never presented as a WebGPU win.
