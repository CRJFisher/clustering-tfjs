The crux · select a cell and the panel reads out THAT cell's runnable code

Click or keyboard-select any grid cell: the code panel re-renders for it, and the permalink updates to match

Why the panel reads resolve_params, not a hand-written snippet

55.8 already resolves each cell's effective params with resolve_params(cell, overrides) — the exact GridParams the worker fits. The panel emits from that same function, so the copy-paste code is the literal call the demo just ran; a hand-typed snippet would silently drift from the picture and break the grid's trust.

The generator · one resolved-params record → the runnable estimator call

Each algorithm's ~5 lines, generated from its resolved params — SOM is the two-call exception

The round-trip · live state ⇄ URL hash, both directions

"Share this result" encodes the state into the hash; on load that hash is decoded and applied — garbage degrades to defaults

Open question this resolves: how much state, and whether to version it

The task left the schema open. The decision: encode only what reproduces the visible result — race n, the selected cell, and the non-Auto ControlOverrides — and version it. Omitting Auto fields keeps a default link short, and a future demo can extend v=2 without breaking v=1 links.

The wire format · the exact hash fields and the central UTM helper

One versioned schema and one link-tagging helper — the minimal state, the minimal diff

The wiring · the UI factories stop returning void so main can orchestrate

make_race_ui and make_grid_ui return small controllers, and main.ts becomes the orchestrator that wires restore + share + the code panel

Why controllers and not a global event bus

Only main.ts needs cross-fold coordination — a narrow typed return per factory is the minimal surface (YAGNI). The 55.8 controls panel gains one method, apply_overrides(): a decoded permalink sets the controls exactly as a user drag would, so the restored grid re-clusters through the existing single-flight path.

The surface area · one pure module, a code generator, controller returns, a CTA fold

A pure permalink module, a per-cell code generator, two controllers, and the Section 3 CTA DOM
