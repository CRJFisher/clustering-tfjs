An n-slider (200 → 5000, hard cap) re-runs the race live, flipping the caption between "CPU wins" and "GPU pulls ahead" and marking the crossover n — a first-class small-n CPU win is what makes the demo un-riggable.

range 200 – 5000 hard cap n = 5000 re-run debounced 280ms, single-flight crossover empirical sign-change bracket builds on 55.4 race UI · 55.3 run_race
Comprehension job 1 · the crux · feel it before reading the code

Race a small n and a large n — the mark appears between your two measurements

What's really new here

55.4 already raced one fixed n=2000 and showed a CPU win as "CPU is faster". 55.5 makes n a live, debounced, single-flight input, flips a plain-language caption, and marks the crossover only where the visitor has empirically bracketed it.

Comprehension job 2 · the physics · why a crossover exists at all

Each lane's wall time is fixed overhead + compute — and the two have opposite shapes

Why the cap is n = 5000

The same n² grows the dense affinity matrix to 25M cells at n=5000, risking OOM/jank on mobile — so the slider hard-caps at 5000 and the controller re-clamps n; compute runs in workers, so large n never freezes the main thread (AC#4).

Comprehension job 3 · the detection · the page marks only what it measured

No model fit — the mark comes from an observed CPU→GPU sign change between two raced n's

Why bracketing, not a least-squares fit — the honesty contract

The page refuses numbers it did not measure — elapsed-only stopwatch, compile excluded, parity as a checksum. A curve fitted to a few noisy medians is an unfalsifiable prediction, so the crossover is marked only where the visitor empirically bracketed it.

Comprehension job 4 · the hazard · dragging fires a flood of input events

One drag = dozens of input events — debounce 280ms, then run latest-n-wins single-flight

Why this stays smooth (AC#4)

Every measured computation runs off-thread in the two workers (55.3, unchanged), so even n=5000 never blocks the main thread; the single-flight guard just prevents two worker-pairs at once, and debounce ties worker churn to landings, not raw events.

As built · what 55.5 added on top of the shipped 55.4 race UI

One pure estimator module; the harness and protocol stayed untouched
