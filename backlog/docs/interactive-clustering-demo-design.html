The headline gap that IS the feature

WebGPU is not wired into the library today (only cpu / webgl / wasm / tensorflow). Adding @tensorflow/tfjs-backend-webgpu behind a per-backend ESM loader (M0) is the one library change worth making — and without it there is no race.

03

One global engine per context → one worker per backend

TensorFlow.js keeps a single global engine per JS context (src/backend/backend.ts is a module-level singleton; Clustering.init calls tf.setBackend once). WebGPU and CPU cannot be active together on the main thread — so a fair concurrent race requires a dedicated Web Worker per backend, each its own JS realm with its own engine. This is the only correct design.

Load-bearing constraint

tfjs has one global engine per JS context — separate workers are not a preference, they are the only way to run two backends concurrently for a fair race.

04

The fairness protocol — honesty as a feature

The demo faces Hacker News scrutiny, so the timing rules are surfaced in the UI, not buried. The same harness runs in every worker.

1
One float32 input, identical across lanes
Never compare a float64 JS path against float32 GPU — the "speedup" would be partly a precision artifact.
2
2–3 discarded warmups per backend
Absorb WGSL shader compile, wasm instantiation, lazy kernel registration before any timed run.
3
Time the full call including the awaited readback
await tensor.data() is inside the timer. Timing only JS dispatch produces a fake ~100× and reads as dishonest.
4
Exactly one readback boundary per run
Identical across lanes, so no lane is advantaged by skipping a transfer.
5
≥5 timed reps; report the median
Plus min/max — never a single best run.
6
Dispose between runs; assert numTensors returns to baseline
tf.memory().numTensors back to baseline so leaks don't skew results.
7
First-run shader-compile time disclosed separately
Behind a toggle, never folded into the headline multiplier.

06

Why Spectral affinity is the headline — not KMeans

Spectral RBF affinity construction (O(n²·d)) is essentially one big matmul/exp block with a single readback before the eigensolver — the cleanest, largest, most defensible gap. KMeans is a trap: its Lloyd loop reads back every iteration and reduces centroids in a pure-JS double loop, stalling the GPU on the bus.

08

Graceful WebGPU fallback — the banner is load-bearing

WebGPU is not clean cross-browser: Chrome/Edge 113+, Safari (macOS/iOS 26), Firefox 141+ Win / 145+ macOS-ARM — Firefox on Linux/Android has none yet, and much social traffic is mobile. So the fallback is core, not an afterthought.
