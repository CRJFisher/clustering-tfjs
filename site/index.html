How this is timed
  • Each backend runs float32 on byte-identical input tensors.
  • The dataset is uploaded to each backend once, outside the timed region — timing measures compute, not host→device transfer.
  • 3 warmup runs are discarded to absorb shader compile / wasm init / lazy kernel registration before any size is timed.
  • Each timed rep brackets the affinity construction plus its single awaited readback (await affinity.data()), so deferred GPU work is counted.
  • The plotted figure is the median of 7 timed reps (protocol floor: ≥ 5).
  • After every rep tf.memory().numTensors must return to baseline — a leak aborts that backend instead of plotting a drifting number.
  • A cross-backend checksum on the affinity matrix confirms both backends computed the same kernel.
