How this is timed
  • Every lane runs float32 on byte-identical input tensors.
  • The dataset is uploaded to each backend once, outside the timed region — timing measures compute, not host→device transfer.
  • 3 warmup runs are discarded to absorb shader compile / wasm init / lazy kernel registration; the cold first run is reported separately (toggle above) and never enters the headline multiplier.
  • Each timed rep brackets the affinity construction plus its single awaited readback (await affinity.data()), so deferred GPU work is counted.
  • The reported figure is the median of 7 timed reps (protocol floor: ≥ 5).
  • After every rep tf.memory().numTensors must return to baseline — a leak aborts the race instead of reporting a drifting number.
  • A cross-lane checksum on the affinity matrix confirms both backends computed the same kernel.
