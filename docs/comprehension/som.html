Self-Organizing Maps (SOM)

A SOM trains a 2D grid of weight vectors by competitive learning, so that neurons sitting next to each other on the grid come to respond to nearby regions of high-dimensional input space — a topology-preserving map you can read at a glance through its U-matrix.

Intuition

Picture a flexible rubber sheet of neurons laid out on a 2D grid — say 5×5 = 25 of them. Each neuron carries a weight vector (a "prototype" or codebook entry) that lives in the same space as your data, exactly like a k-means centroid. Training repeatedly shows the sheet a data point, finds the single neuron whose prototype is closest to it — the Best-Matching Unit (BMU) — and pulls that prototype toward the point.

The twist that separates SOM from k-means: it also drags the BMU's grid neighbors along, by an amount that fades with grid distance. This neighborhood coupling is what makes the final map topology-preserving — adjacent neurons end up representing similar inputs.

Two quantities shrink over training: the learning rate (how hard you pull) and the neighborhood radius (how far the pull spreads on the grid). Early, with a large radius, the whole sheet organizes coarsely — it unfolds and orients to the global shape of the data. Late, with a tiny radius and small learning rate, only the BMU and its immediate neighbors adjust, fine-tuning detail. This anneal-from-global-to-local schedule is why a SOM converges to an organized map rather than a tangled one.

In this library the implementation is mini-batched and vectorized in TensorFlow.js. For each batch it computes BMUs for every sample at once, looks up each BMU's row of a precomputed grid-distance matrix, runs it through a kernel (gaussian / bubble / mexican_hat) to get per-neuron influences, and applies one batched weight update normalized by the summed influence per neuron. Because the grid usually holds far more neurons than the clusters you want, the SOM is a quantizer/organizer, not a final clusterer by itself — cluster() bridges that gap by running agglomerative clustering on the neuron prototypes.

SOM vs. K-Means in one line

K-means moves only the winning centroid toward a point. A SOM moves the winner and its grid neighbors, with the neighbor pull fading by grid distance and shrinking over time. That single difference buys you a map you can plot in 2D.

How it works

  1. Validate params & build schedulers. The constructor (som.ts:58) checks grid_width/grid_height ≥ 1 and fills defaults (topology=rectangular, neighborhood=gaussian, num_epochs=100, learning_rate=0.5, initialization=random, tol=1e-4, mini_batch_size=32). initialize_schedulers (som.ts:90) wraps the learning rate and radius into exponential-decay functions; the default radius scheduler decays from max(grid_width,grid_height)/2 down to 1.
  2. Initialize weights. On first fit, initialize_weights (som_neighborhood.ts:129) builds the [grid_height, grid_width, n_features] weight tensor. random draws uniform values scaled to each feature's [min,max] range; linear / pca place neurons along the first two principal components found by power iteration.
  3. Precompute the grid-distance matrix. create_grid_distance_matrix (som_neighborhood.ts:376) builds a [total_neurons, total_neurons] matrix of pairwise grid distances. Rectangular uses plain Euclidean (r,c) distance; hexagonal applies an offset-coordinate transform (even rows shifted by 0.5 in x, rows spaced by √3/2 in y).
  4. Per-epoch decay & shuffle. Each epoch (som.ts:183) reads the current learning rate and radius from the schedulers, calls validate_neighborhood_params (radius must be > 0; warns if it exceeds the grid), then Fisher-Yates shuffles the sample order with a seeded RNG (shuffle_indices, som.ts:981) so updates are not order-biased.
  5. Find BMUs for a batch. train_epoch (som.ts:230) slices the shuffled data into mini-batches. find_bmu_batch (som_neighborhood.ts:488) computes squared distances from every sample to every neuron via ‖x‖² + ‖w‖² − 2·x·w, takes argMin over neurons, and converts flat indices to [row,col].
  6. Compute neighborhood influence. compute_neighborhood_influence_batch (som_neighborhood.ts:720) gathers each BMU's row from the grid-distance matrix, then applies the chosen kernel to produce an [n_samples, total_neurons] influence matrix.
  7. Batched weight update. update_weights (som.ts:292) computes, per neuron j, Δw_j = lr · (Σ_i h_ij (x_i − w_j)) / (Σ_i h_ij). The differences are broadcast over neurons, scaled by influence, summed over samples, then divided by the per-neuron influence sum (guarded by ε=1e-8 via tf.where, sign-preserving for mexican_hat's negative lobes). The new grid replaces the old tensor.
  8. Track quantization error & check convergence. compute_bmu_distances (som_neighborhood.ts:534) gives each sample's Euclidean distance to its BMU prototype; train_epoch sample-weights the batch means into one epoch error. The loop (som.ts:211) stops early when |prev_error − error| < tol.
  9. Assign final labels. compute_final_labels (som.ts:349) runs find_bmu_batch over all data and stores labels_ = row·grid_width + col for each point. predict() repeats this for new data.
  10. Optional 2-phase clustering. cluster(n_clusters, options) (som.ts:439) flattens the trained grid into [total_neurons, n_features], runs AgglomerativeClustering (default ward/euclidean) to merge neurons into macro-clusters, then maps each point's BMU index to its neuron's cluster label.
  11. Build the U-matrix. get_u_matrix (som.ts:589) returns a [grid_height, grid_width] grid where each cell is the average Euclidean distance from that neuron's prototype to its grid neighbors (8-connected rectangular, 6-connected hexagonal). High values mark cluster boundaries; low values mark coherent regions.

The math

BMU selection

bmu(x) = argminj ‖x − wj‖²

The best-matching unit is the neuron whose prototype wj is closest to input x. Computing the full pairwise distance directly is wasteful, so find_bmu_batch expands the squared norm:

‖x − wj‖² = ‖x‖² + ‖wj‖² − 2·x·wj

The −2·x·wj term is a single matrix multiply over the whole batch; the two squared-norm terms are cheap. argMin over j then picks the winner (som_neighborhood.ts:504-515). The constant ‖x‖² does not change the argmin, so only the sum of ‖wj‖² and the dot product really matter.

Neighborhood kernels h(d, σ)

Given grid distance d from the BMU and current radius σ, the kernel decides how much each neuron is pulled. Three are implemented:

gaussian:    h(d, σ) = exp(−d² / (2σ²))
bubble:      h(d, σ) = 1 if d ≤ σ, else 0
mexican_hat:  h(d, σ) = 2 · (1 − (d/σ)²) · exp(−(d/σ)² / 2)

Gaussian (som_neighborhood.ts:583) is smooth and always positive, peaking at 1 for the BMU itself (d=0). Bubble (:602) is a hard cutoff — full influence inside the radius, nothing outside. Mexican-hat (the Ricker wavelet, :620) is positive at the center but negative at medium grid distances: it adds lateral inhibition, pushing mid-range neurons away from the sample. Its amplitude 2 is hardcoded. Try all three side by side below.

Batch weight update

Δwj = λ · ( Σi hij (xi − wj) ) / ( Σi hij )

For a batch, each neuron j moves by the influence-weighted mean pull toward all batch samples xi, scaled by learning rate λ. hij is the kernel influence of sample i's BMU on neuron j. Dividing by Σi hij normalizes so updates do not depend on batch size — this is the batch-SOM update, distinct from the textbook online rule w += λ h (x − w). The denominator is guarded: if its absolute value drops below ε = 1e-8 it is replaced by ε (via tf.where), which both avoids divide-by-zero and stays sign-preserving so mexican_hat's negative lobes correctly push neurons away (som.ts:305-335).

Exponential decay (λ and radius)

v(t) = vfinal + (vinitial − vfinal) · exp(−t / τ),  τ = num_epochs / 5

Both learning rate and radius follow this schedule across epoch t; decay_rate = 5 sets the time constant τ (som_neighborhood.ts:843). The default final value is vinitial · 0.01 for the learning rate and 1 for the radius (som.ts:117). Large radius early ⇒ coarse global organization; tiny radius late ⇒ local fine-tuning.

U-matrix cell

U(i,j) = (1 / |N(i,j)|) · Σn ∈ N(i,j) ‖wij − wn‖

Each cell is the average Euclidean distance from neuron (i,j)'s prototype to its grid neighbors N(i,j). Ridges (large U) sit between clusters; valleys (small U) are coherent cluster cores (som.ts:589-653). This is the canonical way to see structure on a trained map — explore it below.

Quality & topology metrics

QE = (1 / n) · Σi ‖xi − wbmu(x_i)‖
TE = (1 / n) · Σi 1[ bmu1(xi) and bmu2(xi) are not grid neighbors ]

Quantization error (QE) is the mean distance from each sample to its BMU prototype — how well the map quantizes the data, and the convergence signal (som_neighborhood.ts:534, aggregated at som.ts:284). Topographic error (TE) is the fraction of samples whose two closest neurons are not adjacent on the grid — how well topology is preserved (som.ts:670). Low QE with low TE means an accurate, well-organized map.

Edge cases & scikit-learn parity

No fixed scikit-learn reference. scikit-learn has no SOM estimator, so this implementation follows MiniSom conventions (PCA/linear init, exponential decay, gaussian/bubble kernels) rather than an sklearn API. There is no upstream parity test for SOM itself. Where it does touch the repo's sklearn-parity machinery is the second phase of cluster(): it delegates to AgglomerativeClustering (default ward/euclidean, som.ts:469-475) — the repo's scikit-learn-parity-tested hierarchical clusterer. So SOM inherits sklearn-parity behavior only indirectly, through the agglomerative merge that turns neuron prototypes into final labels. Reproducibility uses the shared seeded RandomStream for both weight init and the per-epoch shuffle.
  • Convergence can stop very early. The tol check compares consecutive epoch quantization errors, so a flat or noisy curve can break the loop well before num_epochs — especially with bubble on small batches where the error is near-constant.
  • Batch-SOM, not classic online SOM. fit() uses an influence-normalized batched update (delta divided by sum of influences). This makes updates batch-size-independent but changes the exact trajectory versus w += λ h (x − w).
  • mexican_hat produces negative influences. The update normalizes by the signed influence sum (guarded with |sum| > ε via tf.where) so neurons in the inhibitory lobe move away from the sample. A near-zero signed sum is replaced by ε to avoid blow-ups.
  • Radius must stay positive. validate_neighborhood_params throws if radius ≤ 0, and the default radius scheduler floors at 1. It only warns when radius exceeds the grid size, which causes near-uniform influence.
  • Grid-distance metric is topology-specific. U-matrix, are_neighbors, and the neighbor-distance matrix all use 8-connectivity (rectangular) or 6-connectivity (hexagonal) and must stay mutually consistent — the source comments explicitly flag this.
  • linear/pca fall back to random when n_samples < 2 or n_features < 2, since power-iteration PCA needs at least two samples/features.
  • Labels are raw BMU indices (0..total_neurons−1), not cluster ids. Treating them as clusters directly is usually wrong; use cluster() for meaningful groupings.
  • partial_fit requires online_mode=true (throws otherwise) and enforces matching n_features. Its virtual_epoch = floor(total_samples_learned / n_samples) drives the same schedulers, so streaming decay depends on how data is chunked.
  • Unfitted access throws. predict on an unfitted model throws ('SOM must be fitted before prediction'); quantization_error(), get_u_matrix(), and get_weights() also throw before fit.
  • PCA init uses power iteration (100 iterations, deflation) because TF.js lacks eigendecomposition; results can differ slightly from a true SVD/eig and from MiniSom's NumPy PCA.
