Agglomerative Clustering

Bottom-up hierarchical clustering that repeatedly merges the two closest clusters via a nearest-neighbor-chain engine with Lance–Williams distance updates, then cuts the resulting merge tree by a target cluster count or a distance threshold.

Intuition

Start with every sample as its own cluster — n singletons. Repeatedly find the two clusters that are closest under a chosen linkage rule, fuse them into one, and record how far apart they were at the moment they merged. Do this n−1 times and you get a full binary merge tree (a dendrogram): its leaves are the original points and its internal nodes are merges, each tagged with a merge height (the distance).

To get a flat partition you cut the tree. Either stop after enough merges that exactly n_clusters groups remain, or stop merging once the next merge would be at or above a distance_threshold. One tree, many possible partitions.

The subtle part is the word "closest". The distance between two single points is obvious, but the distance between two groups needs a linkage rule: single (closest cross-pair), complete (farthest cross-pair), average (mean over all cross-pairs), or Ward (the increase in within-cluster variance from merging).

The clever trick: you never recompute group distances from scratch. When clusters i and j merge into t, the distance from t to any other cluster k is a fixed algebraic combination of the already-known D(i,k), D(j,k), and D(i,j). That recurrence is the Lance–Williams formula, and all four linkages are special cases of it.

A naive engine rescans the shrinking distance matrix on every merge. This implementation instead walks a nearest-neighbor chain: it follows "who is whose nearest neighbor" until it hits a reciprocal pair (two clusters that are each other's nearest neighbor), which is provably a safe merge for "reducible" linkages (single, complete, average, Ward). That guarantees O(n²) time. Because chain-discovery order is not a valid dendrogram order, the merges are sorted by distance before being handed back for cutting.

A second obsession of this codebase is exact scikit-learn parity. Distances are computed in float64 (not the tfjs float32 backend), and the Ward update is arranged to round bit-identically to scipy — so even exactly-tied merge distances on degenerate data (integer grids, duplicate points) resolve the same way sklearn resolves them.

How it works

The pipeline inside fit() (agglomerative.ts:103) runs these stages in order:

  1. Validate params. Enforce exactly one of n_clusters / distance_threshold (XOR), n_clusters a positive integer, distance_threshold > 0, a valid linkage and metric, and that Ward requires euclidean. :220–260
  2. Build the float64 distance matrix D. For precomputed, validate square / symmetric / zero-diagonal and copy into a flat row-major Float64Array. Otherwise materialize the coordinates and compute the full n×n pairwise matrix in float64, computing the upper triangle and mirroring it. :110–141, :269–329
  3. Guard sample count & trivial case. Throw if n_clusters > n_samples (only in n_clusters mode). If n_samples === 1, short-circuit to labels_=[0], empty children_/distances_, n_leaves_=1. :143–154
  4. Build the full merge tree with NN-chain. nn_chain_cluster(D, n_samples, linkage) keeps an active-slot flag array, per-slot cluster sizes, and an int chain. It extends the chain to the current nearest neighbor (preferring the previous chain element on ties) until a reciprocal pair appears. linkage.ts:93–152
  5. Merge & Lance–Williams update. On a reciprocal merge: survivor = max slot, removed = min slot; deactivate removed; for every other active cluster k recompute D(survivor,k) via the linkage recurrence and write both symmetric entries; set the survivor's size; push a merge record. linkage.ts:154–193
  6. Sort merges by distance. NN-chain returns merges.sort() ascending by distance, because discovery order is not a valid dendrogram cut order. linkage.ts:195
  7. Cut the tree. With distance_threshold: keep merges with distance < threshold (filter). Otherwise: keep the first n_samples − n_clusters merges (slice). :159–161
  8. Build sklearn-style children_. build_children replays kept merges through a union-find over 2n−1 nodes, emitting [min,max] node ids; leaves are 0..n−1 and merge m creates internal node n+m. :335–365
  9. Extract flat labels via union-find. A second union-find over n slots replays every kept merge, then each sample's root is mapped to a contiguous label 0..n_clusters−1 in first-seen order. :168–202

The math

Pairwise distances are computed in float64 over the upper triangle and mirrored (agglomerative.ts:269–329). Here x_i[k] is the k-th feature of sample i.

D(i,j) = √( Σk (xi[k] − xj[k])² )

Euclidean (metric='euclidean', the default). Sum the squared per-feature differences, take the square root. :277–291

D(i,j) = Σk |xi[k] − xj[k]|

Manhattan (L1, metric='manhattan'). Sum the absolute per-feature differences. :292–304

D(i,j) = 1 − (xi · xj) / (‖xi‖ · ‖xj‖) ;   = 0 if either norm is 0

Cosine (metric='cosine'). One minus the cosine similarity. A zero-norm vector defines the distance as 0 to avoid dividing by zero. :305–325

The four Lance–Williams updates

When clusters i and j (sizes n_i, n_j) merge into t, the distance from t to another cluster k (size n_k) is recomputed from the three known distances D(i,k), D(j,k), D(i,j) — never from raw points (linkage.ts:43–78).

single    D(t,k) ← min( D(i,k), D(j,k) )
complete  D(t,k) ← max( D(i,k), D(j,k) )
average   D(t,k) ← ( ni·D(i,k) + nj·D(j,k) ) / ( ni + nj )
ward      D(t,k) ← √( max( (ni+nk)·t·D(i,k)² + (nj+nk)·t·D(j,k)² − nk·t·D(i,j)² , 0 ) ),   t = 1/(ni+nj+nk)

single tracks the closest pair (loose chains), complete the farthest pair (compact balls), average (UPGMA) is the size-weighted mean. Ward minimizes the increase in within-cluster variance. Crucially, the factor t = 1/(n_i+n_j+n_k) is distributed into each term rather than dividing a summed numerator once. The two forms are algebraically identical but round differently in IEEE-754 — summing first and dividing last can shift the result by one ULP, which would flip exact distance ties on symmetric data. This arrangement is bit-identical to scipy's _ward. The argument is clamped to 0 before the square root. linkage.ts:59–76

Cutting and node ids

merges_kept = all_merges[ 0 : n_samples − n_clusters ]

n_clusters cut. Keep exactly enough of the distance-sorted merges to leave n_clusters groups. :161

merges_kept = { m ∈ all_merges : m.distance < distance_threshold }

distance_threshold cut. Strict less-than: merges at or above the threshold are not applied; the cluster count is data-determined. :160

node( merge m ) = n_samples + m,   m = 0, 1, 2, …

Internal node id. The sklearn/scipy children_ convention: leaves are 0..n−1, and the m-th merge creates internal node n+m. Each row is emitted as [min,max]. :353–361

Edge cases & scikit-learn parity

  • Single sample (n=1): short-circuits to labels_=[0], empty children_/distances_, n_leaves_=1. :148–154
  • XOR on stopping criterion: providing both or neither of n_clusters and distance_threshold throws "Provide exactly one…". :231–235
  • Ward requires euclidean: any other metric throws. :257–259
  • n_clusters > n_samples throws — but only in n_clusters mode; threshold mode skips this check. :143–145
  • Float64 distance computation is deliberate: matching sklearn's float64 precision (rather than the tfjs float32 backend) is what makes ward/complete/average ties resolve identically to sklearn on degenerate data like integer grids and duplicate points. :124–141
  • Ward distributes the factor per-term to match scipy's _ward bit-for-bit; the algebraically-identical "sum then divide" form can shift one ULP and flip tied merges. linkage.ts:59–76
  • Single-linkage parity caveat: it produces a correct single-linkage tree, but tie-breaking among equally valid merge orders can differ from sklearn/scipy; partitions agree only when there are no exactly-tied distances. :25–29
  • NN-chain tie rule: on distance ties it prefers the previous chain element to avoid cycles, matching scipy's strict-inequality NN-chain. linkage.ts:126–134
  • Merge discovery order is NOT a valid cut order: merges must be sorted by distance before cutting. linkage.ts:85–87, :195
  • Precomputed matrix is validated for square shape, symmetry within 1e-8, and zero diagonal within 1e-8. :371–400
  • Cosine with a zero-norm vector defines distance as 0 to avoid divide-by-zero. :321
The parity promise. Ward, complete, and average linkage reproduce scikit-learn's partitions exactly, ties included, via: (1) float64 pairwise distances matching pairwise_distances; (2) the ULP-stable Ward update; (3) scipy's strict-inequality NN-chain tie rule; (4) sorting merges by distance; and (5) emitting children_ with leaves 0..n−1 and node n+m per merge as [min,max]. Single linkage matches only when no exactly-tied distances exist.
