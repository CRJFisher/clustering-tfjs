Eigendecomposition Engine

A size-adaptive symmetric eigensolver that extracts the k smallest eigenpairs of a normalized graph Laplacian — Jacobi for small matrices, Lanczos for large ones — then deterministically orders and sign-fixes them to match scikit-learn's spectral embedding.

Jacobi n ≤ 100, O(n³), every eigenpair Lanczos n > 100 & k < n/3, O(n²·m), bottom slice only

Intuition

Spectral clustering needs the smoothest coordinates on a graph: the eigenvectors of the normalized Laplacian belonging to its smallest eigenvalues. The smallest eigenvalue is always 0, and its eigenvector is constant — every node gets the same value. The next-smallest eigenvalues belong to vectors that bend gently across the graph, and those gentle bends are exactly where cluster boundaries hide. So the engine exists to extract the bottom slice of the spectrum of a real symmetric matrix — accurately and reproducibly.

There are two regimes:

Jacobi  small matrices (n ≤ 100)

A full eigendecomposition by cyclic Jacobi: repeatedly apply a tiny 2×2 plane rotation that zeros out one off-diagonal pair at a time, slowly grinding the matrix toward diagonal form while accumulating the rotations into an eigenvector matrix V. O(n³), but rock-solid, and it produces every eigenpair.

Lanczos  large matrices (n > 100, k small)

An iterative method that never needs the full matrix — only the ability to multiply it by a vector. Lanczos builds a small tridiagonal matrix whose extreme eigenvalues rapidly approximate the extreme eigenvalues of the big matrix, so you get the bottom k cheaply at O(n²·m).

The subtle part is reproducibility. Eigenvectors are defined only up to sign, and (within repeated eigenvalues) up to rotation; iterative solvers return them in arbitrary order. Downstream clustering must be deterministic, so a post-processing pass sorts eigenpairs ascending by eigenvalue and flips each vector so its largest-magnitude component is positive — exactly scikit-learn's convention. The numerically-wobbly near-constant zero eigenvector is replaced with the exact 1/√n constant vector.

A recurring theme is float32 hygiene. TensorFlow.js works in float32, which erodes orthonormality as rotations accumulate, so the engine re-runs Gram-Schmidt on the eigenvector columns, double-orthogonalizes inside Lanczos when cancellation is detected, and clamps tiny negative eigenvalues to 0 (the Laplacian is known to be positive semi-definite).

How it works

  1. Dispatch by size and shape. smallest_eigenvectors_with_values picks a path: a matrix-free LanczosOperator always goes Lanczos; a dense matrix with n > 100 and k < n/3 goes Lanczos (Jacobi fallback allowed); otherwise Jacobi. LANCZOS_THRESHOLD = 100.
  2. Jacobi path — full decomposition. improved_jacobi_eigen(matrix, {is_psd:true, max_iterations:3000, tolerance:1e-14}). Cyclic sweeps cover all (p,q) pairs, skipping any off-diagonal below an adaptive threshold tolerance · ‖A‖F.
  3. Jacobi rotation math. For each non-negligible pair, compute the rotation that zeros A[p][q], update the affected rows/columns in stable rank-2 form, and accumulate the rotation into V. Equal diagonals use a 45° rotation.
  4. PSD clamp + orthonormality check. With is_psd, negative eigenvalues are clamped to 0 (warning below -tolerance·100). Pairs are sorted ascending and the code verifies max|vi·vj| and |‖vi‖−1| stay under 1e-6.
  5. QR algorithm (alternative dense path). qr_eigen_decomposition iterates Ai+1 = RiQi with a Wilkinson shift for fast convergence on the trailing eigenvalue, accumulating V = Q₀Q₁… If a singular shift makes tf.linalg.qr emit NaN, it retries that step unshifted, then runs Gram-Schmidt on V to repair float32 drift.
  6. Tridiagonal QL solver. tridiagonal_qr_eigen and the Lanczos-internal tridiagonal_ql implement Numerical Recipes tqli: find a negligible off-diagonal, compute a Wilkinson shift from the bottom 2×2 block, then chase the bulge with Givens rotations. Capped at 30 iterations per eigenvalue.
  7. Lanczos — build the Krylov tridiagonal. Start from a seeded random unit vector (seed 42). At step j: w = A·vj; w −= βj-1vj-1; αj = vj·w; w −= αjvj. This produces the symmetric tridiagonal T (α = diagonal, β = off-diagonal).
  8. Full reorthogonalization. After each step, project w against all prior basis vectors. If the norm drops below 0.7071 of its pre-orthogonalization value, run a second Gram-Schmidt pass to fight float32 cancellation. If βj < 1e-10 it's a lucky breakdown (invariant subspace found).
  9. Convergence test + restart. Every 5 steps past j ≥ 2k, solve T and check the residual bound |βm·slast,i| < tol·max(1,|θi|) for the k smallest Ritz values. On reaching m_max without convergence, simple-restart from the best Ritz vector; up to 10 restarts.
  10. Reconstruct Ritz vectors + clamp. extract_smallest rebuilds u = Q·s, clamps negatives to 0 under is_psd (warn below -1e-5), and sorts ascending.
  11. Deterministic ordering + sign fix. deterministic_eigenpair_processing sorts ascending and flips each vector so its largest-magnitude entry is positive. Lanczos applies the same convention inline in extract_smallest.
  12. Select k + zero-eigenvalue padding. Both paths count numerically-zero eigenvalues c (Jacobi TOL 1e-7; Lanczos TOL 1e-2) and return k + c columns, so near-zero eigenvectors from disconnected components aren't dropped. Lanczos over-requests k+5 pairs up front.
  13. Constant eigenvector substitution. create_constant_eigenvector returns the exact unit constant vector (every entry 1/√n), replacing the noisy zero-eigenvalue vector of a connected graph — matching sklearn.

The math

QR iteration

Ai = Qi Ri ,   Ai+1 = Ri Qi  →  diag(λ) ,    V = Q₀ Q₁ … Qi

Each QR step factors A into an orthogonal Q and upper-triangular R, then reverses the product. Because RiQi = QiTAiQi this is a similarity transform — it preserves eigenvalues while pushing mass onto the diagonal. The accumulated product of all Q columns becomes the eigenvectors.

Wilkinson shift

δ = (a − c)/2 ,   μ = c − sign(δ)·b² / (|δ| + √(δ²+b²))

From the trailing 2×2 block [[a,b],[b,c]] (here a = A[n-2][n-2], b = A[n-2][n-1], c = A[n-1][n-1]), the shift μ is the eigenvalue nearest c. Subtracting μ·I before each QR step gives near-cubic convergence on the trailing eigenvalue. (qr.ts:51, lanczos.ts:536)

Jacobi rotation

θ = (aqq − app)/(2 apq) ,   t = sign(θ)/(|θ| + √(1+θ²)) ,   c = 1/√(1+t²) ,   s = t·c

This plane rotation annihilates the (p,q) off-diagonal entry. θ measures how far the two diagonal entries are apart relative to their coupling apq; t = tan of the rotation, from which the cosine c and sine s follow. The auxiliary τ = s/(1+c) lets the row/column updates be written in numerically stable rank-2 form. (improved.ts:105)

Lanczos three-term recurrence

βj vj+1 = A vj − αj vj − βj−1 vj−1 ,   αj = vjT A vj

Each new basis vector is A·vj with the components along the previous two basis vectors subtracted off, then normalized — the norm of the leftover is βj. The resulting orthonormal basis Q satisfies QTA Q = T, a small tridiagonal matrix (α on the diagonal, β off it). The eigenvalues of T — the Ritz values — approximate the extreme eigenvalues of A. (lanczos.ts:127–188)

Ritz convergence bound

‖A ui − θi ui‖ ≈ |βm · sm,i| < tol · max(1, |θi|)

A converged Ritz pair (θi, ui = Q si) has a residual equal to the last component of T's eigenvector si scaled by the final βm. This is cheap to evaluate without ever forming the full vector ui — you peek at the bottom entry of the tiny tridiagonal eigenvector. (check_convergence, lanczos.ts:362)

Ritz vector reconstruction

ui = Q · si = Σcol basis[col] · s[col][i]

An eigenvector of the big matrix is a linear combination of the small Lanczos basis vectors, weighted by the tridiagonal eigenvector. (extract_smallest, lanczos.ts:432)

Sign convention

r* = argmaxr |v[r]| ;   if v[r*] < 0 then v ← −v

Find the entry of largest absolute value and force it positive. Since an eigenvector is only defined up to ±1, this makes identical input yield identical embeddings — and it matches scikit-learn. (post.ts:71–84, lanczos.ts:442)

Constant eigenvector

v₀ = (1/√n) · 𝟙    (every entry equal, ‖v₀‖ = 1)

The exact null-space vector of a connected normalized Laplacian. The plain 1/√n constant is substituted for the noisy numerical one — note this is the plain constant, not the degree-weighted form. (constant_eigenvector.ts:23)

Off-diagonal convergence norm

off(A) = √( Σi≠j A[i][j]² )

The Frobenius norm of the off-diagonal entries. Jacobi and QR iterate until this falls below tolerance, signaling the matrix is effectively diagonal. (qr.ts:34, improved.ts:55)

Edge cases & scikit-learn parity

SituationHandling
Sign ambiguityEigenvectors are defined up to ±1; both paths force the largest-magnitude component positive (post.ts:71, lanczos.ts:442) for determinism.
Degenerate / repeated eigenvaluesEigenvectors are defined only up to rotation within the eigenspace; sorting is stable on value, but the basis inside a degenerate block is solver-dependent.
Constant eigenvectorThe connected-graph zero eigenvector is numerically noisy; create_constant_eigenvector substitutes the exact 1/√n vector — the plain constant, not the degree-weighted version.
PSD clampingTiny negative eigenvalues from float32 are set to 0; magnitudes beyond tolerance (Jacobi −tolerance·100, Lanczos −1e-5) log a warning rather than silently corrupting results.
Zero-eigenvalue paddingBoth paths return k + c columns (c = count of near-zero eigenvalues) so disconnected-component eigenvectors aren't dropped. Counting tolerances differ (Jacobi 1e-7, Lanczos 1e-2 in the path, 1e-7 in laplacian_eigen_decomposition).
QR singular shiftWhen the Wilkinson shift hits an exact eigenvalue, A_shifted is singular and tf.linalg.qr can return NaN on some platforms; qr.ts:90 detects non-finite R and retries that step unshifted.
Float32 orthonormality driftAccumulated rotations erode orthogonality, so QR re-runs gram_schmidt_columns and Jacobi validates with a 1e-6 post-condition check.
Lanczos cancellationFull reorthogonalization plus a conditional second pass guard against loss of orthogonality in the Krylov basis.
Lucky breakdown, too few pairsLanczos generates a random vector orthogonal to the current basis (up to 5 attempts) to continue; returns early if the basis already spans the space.
Non-convergenceQR, Jacobi, tridiagonal QL (30-iter cap) and Lanczos (max_restarts) all warn and return best-effort results rather than throwing.
Lanczos failurelanczos_path catches errors and falls back to Jacobi only for dense matrices (not matrix-free operators) and only when allowed.

scikit-learn parity

The post-processing convention is explicitly modeled on scikit-learn: ascending eigenvalue order plus sign-fixing so each eigenvector's largest-magnitude component is positive. The constant-eigenvector substitution mirrors sklearn's spectral_embedding, which replaces the noisy zero-eigenvalue vector for connected graphs with a clean constant — and crucially uses the plain 1/√n constant rather than the degree-weighted form. PSD clamping reflects sklearn's assumption that the normalized Laplacian's spectrum lies in [0, 2], so small numerical negatives are zeroed. The deterministic random_seed=42 Lanczos start makes the iterative path as reproducible as the dense path. The k + c zero-eigenvalue padding parallels how connected-component count affects the usable embedding dimension. The tridiagonal QL solver follows Numerical Recipes tqli (Golub & Van Loan §8.3.3) — the same algorithm family LAPACK/sklearn rely on under the hood.
