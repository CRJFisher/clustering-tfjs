K-Means Clustering

Partition n points into K clusters by alternately assigning each point to its nearest centroid and recomputing centroids as cluster means — restarted from several k-means++ seedings, keeping the run with the lowest inertia.

Lloyd's iteration k-means++ (greedy) n_init restarts sklearn parity

Intuition

Imagine dropping K flags onto a scatter of points. Every point joins the nearest flag, forming K groups. Then each flag walks to the center of mass of its group. Points may now be closer to a different flag, so they re-join, and the flags move again. Repeat. This is Lloyd's iteration, and it monotonically decreases a single cost — the inertia, the total squared distance from points to their assigned centroid. It always converges, but only to a local minimum, so where you start matters a lot.

The starting positions come from k-means++, a smarter seeding than picking K random points. It picks the first center uniformly at random, then picks each subsequent center with probability proportional to its squared distance from the nearest center already chosen. This spreads the seeds apart, dramatically reducing the chance of two landing in the same blob. To be more robust still, this implementation uses the greedy variant: at each step it samples several candidate centers and keeps the one that most reduces the overall potential.

Because Lloyd only finds a local optimum, the whole procedure is run n_init times (default 10) from independent k-means++ seedings; the result with the lowest final inertia wins. This is the standard scikit-learn recipe and the main lever for stable, reproducible results.

Two precision details in this code. k-means++ probabilities and centroid means are computed in full-precision JavaScript arrays (the original number[][] data), while the heavy assignment-distance step runs on float32 tensors via a matrix-multiply expansion of squared distance. This split keeps seeding faithful to scikit-learn while still using fast tensor math in the inner loop.

How it works

  1. Validate & ingest. validate_params checks n_clusters ≥ 1, max_iter ≥ 1, tol ≥ 0, n_init ≥ 1. fit rejects empty data and K > n_samples before any tensor is created. Input is cloned to a float32 x_tensor, and a full-precision points_arr is kept for seeding and means.
  2. Restart loop. run_once runs n_init times with seed_offset = run index. If random_state is set, the per-run seed is base_seed + seed_offset — deterministic but distinct seedings. The lowest-inertia run is kept; the rest are disposed.
  3. k-means++ first center. A NumPy-aligned random stream is created; the first centroid index is rand_int(n_samples), added to centroid_idxs and a membership Set.
  4. k-means++ greedy seeding. While fewer than K centers: compute each point's squared distance to its nearest chosen center (0 if it is one); sum to current_pot. Draw local_trials = 2 + ⌊ln K⌋ candidate indices by inverse-CDF sampling (binary search over the cumulative distance array with rand()·current_pot). For each candidate compute the resulting total potential and keep the candidate giving the lowest potential.
  5. Degenerate-potential guard. If current_pot === 0 (all remaining points coincide with existing centers), deterministically append the first unused index instead of sampling — preventing division-by-zero / NaN.
  6. Assignment (E-step). Distances are computed as ‖x‖² + ‖c‖² − 2·x·cᵀ on float32 tensors, clamped to ≥ 0. argMin(1) gives the labels; min(1) summed gives this iteration's inertia.
  7. Update (M-step). Accumulate per-cluster feature sums and counts over points_arr (double precision), then divide by counts for the new centroids.
  8. Empty-cluster handling. A cluster with count 0 first keeps its old centroid, then is relocated: points are sorted by descending distance-to-nearest-center and the farthest points reseed the empty clusters, one farthest point per empty cluster.
  9. Convergence check. Compute centroid_shift = max|old−new| and relative inertia change |prev−curr|/prev. Break when either falls within tol; otherwise iterate up to max_iter.
  10. Persist best. After all restarts, store centroids_, labels_ (as number[]), inertia_; dispose x_tensor. fit_predict awaits fit and returns labels_.

The math

Everything K-Means does is in service of minimising one objective.

Inertia (objective)
J = Σi ‖ xi − ca(i) ‖²,   a(i) = argmink ‖ xi − ck ‖²

The total squared Euclidean distance from each point xi to its assigned centroid ca(i). Here a(i) is the cluster index assigned to point i. Lloyd's iteration drives J downward; across restarts the run with the smallest J is kept.

Assignment — E-step
a(i) ← argmink ‖ xi − ck ‖²

Every point joins its nearest centroid. In code this is an argMin over the point-to-centroid distance matrix.

Distance via matmul expansion
‖ xi − ck ‖² = ‖ xi ‖² + ‖ ck ‖² − 2 · xi · ckᵀ,   then max( · , 0)

Expanding the squared norm avoids materialising a giant pairwise-difference tensor: the only heavy operation is the matrix product X·Cᵀ. The max(·, 0) clamp removes tiny negative values produced by float32 round-off, which would otherwise corrupt argMin.

Update — M-step
ck ← ( 1 / |Sk| ) · Σi ∈ Sk xi

Each centroid becomes the mean of its assigned points, where Sk is the set of points assigned to cluster k and |Sk| is its count. This is the step that "walks the flag to the center of mass."

k-means++ sampling
P( choose xi ) ∝ D(xi)²,   D(xi) = minc chosen ‖ xi − c ‖

The probability of selecting a point as the next seed grows with the square of its distance to the nearest already-chosen center — so the next seed tends to land far from existing seeds, spreading initial centers across the data.

Greedy candidate count
local_trials = 2 + ⌊ ln K ⌋

Rather than committing to the first sampled candidate, the greedy variant draws this many candidates per step and keeps the single one that minimises the total potential J. This matches scikit-learn's n_local_trials.

Convergence test
stop if  | Jprev − J | / max( Jprev, 1 ) ≤ tol  OR   maxk ‖ ck − ck′ ‖∞ ≤ tol

Either a negligible relative inertia change or a negligible absolute centroid movement halts iteration. A run can converge on centroid stability even while inertia is still nudging.

The inner loop, verbatim

for (let iter = 0; iter < max_iter; iter++) {
  const distances = tf.tidy(() => {
    const x_norm = x_tensor.square().sum(1).reshape([n_samples, 1]);
    const c_norm = centroids.square().sum(1).reshape([1, K]);
    const cross  = tf.mat_mul(x_tensor, centroids.transpose());
    const dist_sq = x_norm.add(c_norm).sub(cross.mul(2));
    return tf.maximum(dist_sq, tf.scalar(0, 'float32'));   // clamp round-off
  });
  labels  = await distances.argMin(1).data();              // E-step
  inertia = sum( await distances.min(1).data() );          // objective J
  // ... accumulate sums/counts over points_arr (double precision) → M-step
  // ... relocate empty clusters to farthest points
  const relative_diff = Math.abs(prev_inertia - inertia) / (prev_inertia || 1);
  if (relative_diff <= tol || centroid_shift <= tol) break; // convergence
}

Edge cases & scikit-learn parity

Edge cases handled

  • Empty clusters are never dropped. Each is relocated to one of the points farthest from its nearest centroid (sorted by descending min-distance), one farthest point per empty cluster.
  • Degenerate k-means++ potential. If every remaining point coincides with a chosen center (current_pot === 0), the next center is the first unused index, chosen deterministically — no division-by-zero / NaN.
  • Float32 round-off. Negative squared distances are clamped with max(dist_sq, 0) before argMin / min.
  • Precision split. k-means++ distances and centroid means use full-precision number[][]; only the inner assignment uses float32 tensors — deliberate, to keep seeding probabilities faithful to scikit-learn.
  • Fail fast, no leaks. K > n_samples and empty input both throw before any tensor is created.
  • Tensor hygiene. Losing restarts' centroids, intermediate distance / min / shift tensors, and x_tensor are all disposed; only the best centroids tensor survives on the instance.

scikit-learn parity

Targets scikit-learn KMeans defaults: n_init=10, max_iter=300, tol=1e-4, and greedy k-means++ with local_trials = 2 + ⌊ln K⌋ (sklearn's n_local_trials). The random stream is explicitly NumPy-aligned (make_random_stream) so seeded runs reproduce sklearn's draw sequence; per-restart seeds advance as base_seed + run index. Probabilities and means are computed in double precision to match sklearn's numerics rather than float32. Lowest-inertia-across-restarts selection mirrors sklearn. The matmul distance expansion mirrors sklearn's euclidean_distances trick, including the negative-clamp.

Two intentional/known deviations. (1) The empty-cluster strategy here reassigns the globally farthest points, whereas sklearn relocates points from the most-populous clusters. (2) The convergence test uses a relative inertia threshold and an absolute ∞-norm centroid shift, while sklearn scales tol by the per-feature variance of the data and tests the squared center shift.
