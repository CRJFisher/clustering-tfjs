Choosing k (Model Selection)

Sweep candidate cluster counts k, score each clustering with validation metrics or an inertia-elbow knee detector, and pick the best k.

Intuition

Most clustering algorithms — k-means, spectral, agglomerative, SOM — make you commit to the number of clusters k before they run. But the "right" k is rarely known in advance. find_optimal_clusters turns the question into a search: fit the algorithm for every k in a range, measure how good each clustering is, and let the scores rank the candidates. The whole module is that search loop plus the scoring rules.

There are two distinct mental models, and the module supports both behind one knob called method.

① Metric-based selection

For each k, compute one or more validation indices — silhouette, Davies-Bouldin, Calinski-Harabasz. Because these live on incomparable scales, the 'combined' method normalizes each into [0,1] and averages them so no single metric dominates. 'silhouette' just uses the raw silhouette. Higher combined score wins.

② The elbow / knee

Within-cluster sum of squares (WSS, a.k.a. inertia) always falls as you add clusters — more centroids means every point sits closer to its own. The curve drops steeply, then flattens. The elbow is the k where another cluster stops paying off. Kneedle finds it geometrically.

The geometric trick of Kneedle: normalize the WSS-vs-k curve into a unit box, draw a straight chord from the first point to the last, and find the point that bulges farthest from that chord. For a decreasing (concave) curve the curve sags below the chord, so the gaps are negative — the most-negative gap is the elbow. The selection code negates those differences so the elbow earns the highest combined_score, then sorts descending. That is the same ranking machinery the metric-based methods use: every method ends as a list of combined_score values, sorted, with index 0 the winner.

How it works

  1. Validate range and coerce input. Require min_clusters ≥ 2 and max_clusters ≥ min_clusters. Convert the DataMatrix to a tf.Tensor2D if it isn't already one. Set effective_max_clusters = min(max_clusters, n_samples − 1) so you never request more clusters than points-minus-one.
  2. Decide which metrics to compute. Flags compute_silhouette / compute_db / compute_ch / should_compute_wss are derived from method and metrics. 'silhouette' computes only silhouette; 'combined' computes whichever metrics are listed; 'elbow' computes WSS only; a custom scoring_function computes whatever metrics it lists. No paying for metrics you won't use.
  3. SOM special case (train once). If algorithm === 'som', a single map is trained before the loop — grid sized via the ~5·√n neuron heuristic, at least √(effective_max_clusters). Each k is then produced by phase-2 agglomerative grouping of the trained neuron weights via shared_som.cluster(k). Every other algorithm builds a fresh clusterer per k.
  4. Sweep k and fit. For k from min_clusters to effective_max_clusters: build the clusterer with n_clusters = k plus algorithm_params, then labels = await clusterer.fit_predict(data_tensor). For SOM, labels = shared_som.cluster(k).
  5. Score each clustering. Defaults: silhouette = 0, davies_bouldin = Infinity, calinski_harabasz = 0. The guard has_enough_clusters = new Set(labels).size ≥ 2 catches collapsed labelings; if violated, worst-case values are assigned (silhouette −1, DB Infinity, CH 0) so that k ranks last instead of crashing the metrics.
  6. Read or compute WSS for the elbow. When should_compute_wss is true: if a KMeans instance exposes a non-null inertia_, reuse it directly; otherwise call compute_wss(data_tensor, labels), which gathers each cluster's rows, takes the centroid as the column mean, squares the deviations, and sums across all clusters.
  7. Apply the scoring rule (post-loop). A custom scoring_function already set combined_score per-eval from raw values. Otherwise: 'combined' calls normalize_and_score_evaluations; 'silhouette' copies the raw silhouette into combined_score; 'elbow' calls find_knee on (k, WSS) and sets combined_score[i] = −differences[i].
  8. Kneedle knee detection. find_knee returns null when n < 3 or the x/y range is zero. Otherwise it normalizes both axes to [0,1], computes per-index the chord height y_line and the gap y_norm[i] − y_line, then scans interior indices 1…n−2 for the maximum deviation (negated for concave). A knee is accepted only if best_deviation ≥ sensitivity/(n−1); otherwise knee_x is null.
  9. Elbow fallback. If find_knee returns knee_x === null, the elbow path falls back to combined_score = −k for every evaluation, favoring the smallest k (parsimony) after the descending sort.
  10. Rank, pick optimal, clean up. Sort evaluations by combined_score descending; optimal = evaluations[0]. The input tensor is disposed only if find_optimal_clusters created it; per-k clusterers are disposed inside the loop, the shared SOM after it.

The first and last k can never be the knee — the scan runs only over interior indices 1…n−2, and the chord passes exactly through the endpoints (gap = 0 there). The elbow is, by construction, an interior point.

The math

Within-cluster sum of squares (WSS / inertia)

WSS = Σc Σx ∈ Cc ‖x − μc‖²   μc = (1/|Cc|) Σx ∈ Cc x

Sum over clusters c of the squared Euclidean distances from each point x in cluster Cc to that cluster's centroid μc. In compute_wss, μc is cluster_data.mean(0) and the accumulation is diff.square().sum(). WSS is monotonically non-increasing in k, which is precisely why you can't pick k by minimizing it — you need the elbow, not the minimum.

Min-max normalization — Calinski-Harabasz (higher better)

nch = (ch − chmin) / (chmax − chmin)

A linear rescale of the CH index into [0,1] across all tested k. If the range is 0 or non-finite, nch = 0.5; if CH is Infinity (perfect separation, WSS = 0) then nch = 1.

Min-max normalization — Davies-Bouldin (lower better)

ndb = 1 − (db − dbmin) / (dbmax − dbmin)

DB is inverted because lower is better — small DB becomes a large normalized score. A zero/non-finite range gives 0.5; a non-finite db gives 0 (the worst).

Silhouette normalization (fixed range)

ns = (s + 1) / 2

Silhouette already lives in [-1, 1], so it uses a fixed mapping to [0,1] rather than data-dependent min-max. This keeps its scale comparable across runs instead of shifting with whatever min/max happened to appear.

Combined score

combined_score = (1/m) Σmetric normalized_metric

An unweighted mean of the m selected normalized metrics, giving each equal contribution. An empty metric set yields 0.

Kneedle normalization

x_norm = (x − xmin) / (xmax − xmin)   y_norm = (y − ymin) / (ymax − ymin)

Maps both axes of the WSS-vs-k curve into the unit square so the chord deviation is scale-free — the elbow doesn't depend on whether WSS is in the thousands or the millions.

Difference curve (deviation from chord)

di = y_normi − [ y_first + (y_last − y_first)·(x_normi − x_first)/(x_last − x_first) ]

The vertical gap between the normalized curve and the straight chord joining the first and last points. The bracketed term is y_line at index i. For a decreasing concave curve every interior di is negative.

Knee = extremum of the difference curve

knee_index = argmax1 ≤ i ≤ n−2 ( −di if concave, di if convex )

The interior point of maximum deviation from the chord. For concave (decreasing WSS) the curve sags below the chord, so the most-negative di — equivalently the largest −di — is the elbow.

Significance threshold

knee accepted ⟺ best_deviation ≥ sensitivity / (n − 1)

Scales the minimum acceptable bulge by the average normalized step size 1/(n−1). A higher sensitivity demands a more pronounced knee; failing the test returns knee_x = null.

Elbow → score mapping

combined_scorei ← −di   (fallback: combined_scorei ← −ki)

Negating the difference makes the elbow (most-negative di for concave curves) the highest score, so the standard descending sort selects it. With no significant knee, −k favors the smallest k.

Edge cases & scikit-learn parity

Edge cases

  • find_knee returns no knee (knee_x = null) when n < 3, when x_range = 0, or when y_range = 0; the elbow path then falls back to combined_score = −k (smallest k wins).
  • A knee candidate is rejected if best_deviation < sensitivity/(n−1), again triggering the −k parsimony fallback.
  • Only interior indices 1…n−2 can be the knee; the first and last k are never selected as elbows by construction.
  • Degenerate labelings where fewer than 2 distinct clusters appear (has_enough_clusters false) get worst-case metrics (silhouette −1, DB Infinity, CH 0) so that k sorts last instead of throwing — primarily a SOM phase-2 collapse safeguard.
  • Calinski-Harabasz = Infinity (WSS = 0, perfect separation) normalizes to nch = 1; non-finite DB normalizes to ndb = 0; a zero or non-finite normalization range maps that metric to a neutral 0.5.
  • effective_max_clusters < min_clusters throws "Not enough samples …"; min_clusters < 2 and max_clusters < min_clusters throw on entry.
  • WSS is reused from KMeans.inertia_ when available (and non-null) to avoid an O(n·d) recompute; other algorithms always call compute_wss.
  • Tensor ownership: the input tensor is disposed only if find_optimal_clusters created it from a number[][]; an externally supplied tensor is left intact.

scikit-learn parity

This module is a higher-level convenience layer with no direct scikit-learn equivalent — sklearn has no single find_optimal_clusters — so parity is about the building blocks rather than this API. The validation indices it calls (silhouette_score, davies_bouldin, calinski_harabasz) are the parts held to exact sklearn parity, and the per-k clusterers (KMeans, SpectralClustering, AgglomerativeClustering) carry their own parity guarantees. The Kneedle implementation is a deliberately simplified variant of Satopaa et al. 2011 (normalize, chord-difference, max deviation) rather than the full kneed library — it omits smoothing splines and the online polygon/threshold logic, using a single global maximum of the difference curve with a sensitivity/(n−1) acceptance gate. WSS matches sklearn's KMeans inertia_ definition (sum of squared distances to cluster centroids) and is read directly from inertia_ when KMeans provides it, ensuring numerical consistency with the fitted model.
