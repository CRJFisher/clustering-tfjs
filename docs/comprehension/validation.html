Validation Metrics: Internal and External Cluster Quality

Five scores that grade a clustering — three judge geometry without ground truth, two compare a labeling against a known truth via a contingency table.

Intuition

There are two fundamentally different questions you can ask about a clustering.

The first is "does this clustering look good on its own terms?" Are points tight within their cluster and far from other clusters? You answer this using only the data X and the labels, with no reference answer. These are the internal metrics: silhouette, Davies-Bouldin, and Calinski-Harabasz. They all reward compactness (points near their own centroid/neighbors) and separation (clusters far apart) — but each packages those two ideas differently. Silhouette works per-point and is bounded in [−1, 1]; Davies-Bouldin averages a worst-case "confusability" ratio per cluster and is unbounded with lower=better; Calinski-Harabasz is a single variance ratio (between-cluster spread over within-cluster spread) with higher=better.

The second is "does this clustering agree with a known ground-truth labeling?" You answer this by cross-tabulating the two labelings into a contingency table nij = how many points are in true-class i AND predicted-cluster j. Every external metric here is a function of that one table plus its row/column margins. The key subtlety is chance correction: if you randomly threw points into k buckets you would get some accidental agreement, so a good metric subtracts off what chance alone would produce. ARI does this by counting agreeing pairs and subtracting an expected pair count; NMI measures shared information (entropy) between the two labelings and normalizes it to [0, 1].

Both contingency-based metrics are invariant to how you name or permute the labels — only the partition structure matters. ARI asks "for every pair of points, do the two labelings agree on whether they are together or apart?" NMI asks "knowing the predicted cluster, how much uncertainty about the true class is removed?"

A recurring theme across the whole file set is exact scikit-learn parity: every degenerate case — a singleton cluster, identical points, a zero denominator — is handled to match what sklearn returns, even when the "natural" math would give NaN or Infinity.

How it works

  1. Validate and convert inputs (all metrics). validate_labels_length checks labels.length equals the number of data rows. convert_validation_inputs returns the data as a tf.Tensor2D plus a plain integer label_array (tensor labels are dataSync'd and Math.round'd). An owns_tensor flag records whether this function created the tensor and must dispose it. Internal metrics require k = unique labels > 1; Calinski-Harabasz also requires k < n.
  2. Silhouette — full pairwise distance matrix. Compute D[i,j] = ‖xi − xj‖ via the squared-norm identity x_norm + x_normᵀ − 2·X·Xᵀ, clamped at 0 with tf.maximum before sqrt to avoid negative round-off under the root. Flatten to a Float32Array for the per-sample loops.
  3. Silhouette — per-sample a and b. For each point i, partition the others by label. a(i) = mean distance to same-cluster points. For every other cluster compute the mean distance to its members; b(i) = the smallest such mean (the nearest competing cluster). Score s = (b − a) / max(a, b). Degenerate handling: singleton cluster → 0; a==0 and b==0 (identical points) → 0. silhouette_score is the plain mean of all s.
  4. Davies-Bouldin — centroids and dispersions. For each cluster compute its centroid (mean of points) and dispersion si = mean Euclidean distance from its points to the centroid. A singleton cluster gets dispersion 0.
  5. Davies-Bouldin — per-cluster worst similarity. For each cluster i, over all j≠i compute Rij = (si + sj) / dij where dij is the centroid-to-centroid distance; take the max over j. DB = mean of those maxima over k clusters. Lower is better. Coincident centroids (dij==0): if the dispersion sum > 0 → that cluster's max becomes Infinity (break); if both dispersions are 0 → skip the pair (mirrors sklearn nanmax of 0/0=NaN).
  6. Calinski-Harabasz — BSS and WSS. Compute the global centroid (mean of all points). For each cluster, WSS accumulates the sum of squared distances from points to their cluster centroid; BSS accumulates cluster_size · ‖cluster_centroid − global_centroid‖². Score = (BSS/(k−1)) / (WSS/(n−k)). Higher is better.
  7. External metrics — build the contingency table. build_contingency_table maps each labeling's unique values to dense 0-based indices, then fills table[i][j] = count of points with (true=i, pred=j). It also returns row_sums (ai, true-class sizes), col_sums (bj, pred-cluster sizes), and n.
  8. ARI — pair counts and chance correction. Using C(x)=x(x−1)/2: index = Σij C(nij); sum_a = Σi C(ai); sum_b = Σj C(bj). expected = sum_a·sum_b / C(n); max_index = (sum_a + sum_b)/2. ARI = (index − expected)/(max_index − expected). If C(n)==0 (≤1 sample) → 1.0. If the denominator==0 → return 1.0 when index==expected, else 0.0.
  9. NMI — entropies, mutual information, normalize. H(U) from row_sums and H(V) from col_sums using −p·ln(p) (natural log, skipping zero counts). MI = Σij (nij/n)·ln(n·nij / (ai·bj)) over positive cells. The normalizer is chosen by the average option: arithmetic (H_U+H_V)/2, geometric √(H_U·H_V), min, or max. NMI = MI/normalizer. If normalizer==0 → 1.0 when MI==0, else 0.0.

The math

Silhouette coefficient

s(i) = ( b(i) − a(i) ) / max( a(i), b(i) ) ∈ [−1, 1]

a(i) = mean intra-cluster distance; b(i) = the minimum over other clusters of the mean distance to that cluster. The score is +1 when the point sits far from all rivals, 0 on a boundary, and negative if the point is likely misassigned. The overall silhouette_score = mean_i s(i).

Pairwise distance identity

D[i,j] = √( max(0, ‖xi‖² + ‖xj‖² − 2·xiᵀxj) )

Vectorized Euclidean distance via the Gram matrix X·Xᵀ. The max(0, ·) guards against tiny negative values produced by floating-point round-off before the square root — without it, two coincident points could yield √(−1e-7) = NaN.

Davies-Bouldin index

DB = (1/k) · Σi maxj≠i ( si + sj ) / dij

si = mean distance of cluster i's points to its centroid (its dispersion); dij = distance between centroids i and j. Each cluster contributes its worst (most confusable) neighbor ratio: a cluster that is both fat and close to another fat cluster scores high. Lower = better separated.

Calinski-Harabasz (variance ratio)

CH = ( BSS / (k−1) ) / ( WSS / (n−k) ),   BSS = Σi ni·‖ci − c‖²,   WSS = Σi Σx∈Ci ‖x − ci‖²

Ratio of between-cluster dispersion (cluster centroids ci spread around the global centroid c, weighted by size ni) to within-cluster dispersion (point spread around their own centroid), each scaled by its degrees of freedom. Higher = tighter, better-separated clusters.

Adjusted Rand Index

ARI = ( Σij C(nij) − E ) / ( (Σi C(ai) + Σj C(bj))/2 − E ),   E = Σi C(ai) · Σj C(bj) / C(n)

C(x) = x(x−1)/2 counts the pairs inside a group of x points. The numerator is agreeing pairs minus the chance expectation E; the denominator is the maximum possible minus E. ARI = 1 is perfect, ≈ 0 is random, and it can go negative when agreement is worse than chance.

Mutual information

MI(U,V) = Σij ( nij/n ) · ln( n·nij / ( ai·bj ) )

Information shared between the two labelings; only positive cells contribute. It is zero exactly when the labelings are statistically independent (nij = ai·bj/n for all cells).

Normalized Mutual Information

NMI = MI(U,V) / normalizer,   normalizer ∈ { (H_U+H_V)/2, √(H_U·H_V), min(H_U,H_V), max(H_U,H_V) }

H_U = −Σi (ai/n)·ln(ai/n) and H_V likewise from bj are the entropies of the two labelings. Normalizing MI by an entropy average rescales agreement into [0, 1]; the choice of average is the average option (default arithmetic, matching sklearn).

Edge cases & scikit-learn parity

Every degenerate branch is written to reproduce scikit-learn's exact return value rather than the raw mathematical result.

SituationBehavior
k ≤ 1Silhouette, Davies-Bouldin and Calinski-Harabasz all throw requires at least 2 clusters. Calinski-Harabasz additionally throws when k ≥ n.
Singleton clusterSilhouette returns 0 for that point (no same-cluster neighbors); Davies-Bouldin assigns dispersion 0 to single-point clusters.
Identical points (silhouette)When a==0 and b==0 the formula 0/0 is undefined; the code returns 0 to match the sklearn convention.
Coincident centroids (Davies-Bouldin)dij==0 with a positive dispersion sum forces that cluster's contribution to Infinity; with both dispersions 0 the pair is skipped (sklearn nanmax of 0/0=NaN).
Distance round-offSilhouette clamps the squared-distance expression with max(0, ·) before sqrt so floating-point negatives never produce NaN.

ARI zero denominatorWhen max_index==expected, return 1.0 if index==expected (perfect agreement), else 0.0.
