Deterministic RNG (Mersenne Twister MT19937)

A 32-bit MT19937 pseudo-random generator that byte-for-byte reproduces NumPy’s legacy RandomState stream, giving clustering-tfjs the deterministic, scikit-learn-matching randomness that k-means++ seeding, SOM shuffling, and Lanczos initialization depend on.

Intuition

A pseudo-random generator is a deterministic machine: feed it the same starting seed and it always emits the exact same sequence of “random” numbers. MT19937 (the Mersenne Twister, period 219937−1) holds 624 words of 32-bit internal state. Numbers come out 624 at a time. When the buffer runs dry, a twist step churns the whole state vector forward; each individual draw then runs through a temper step that scrambles the bits of one state word so the output looks statistically uniform even though the underlying state evolves in a simple, structured way. Think of twist as reshuffling the whole deck and temper as a final card-by-card scramble before dealing.

Why does this matter for a clustering library? Algorithms like k-means++ and SOM are randomized: k-means++ picks its first center uniformly at random, then samples each subsequent center with probability proportional to squared distance; SOM shuffles its training rows every epoch. If the randomness is not reproducible you cannot get the same clusters twice, you cannot write deterministic tests, and you cannot match a reference implementation. scikit-learn’s randomized routines are driven by NumPy’s legacy RandomState, which is MT19937. So to produce identical cluster labels to scikit-learn for a given random_state, this codebase reproduces NumPy’s MT19937 bit-for-bit: the same seeding recurrence, the same tempering masks, the same 53-bit float construction, and the same rejection-sampling scheme for bounded integers.

The consumer side is a tiny RandomStream interface — rand() for a float in [0,1) and rand_int(max) for an integer in [0,max) — backed by either MT19937 (when a seed is given) or Math.random (when no seed is given). Seeded means deterministic and parity-checked; unseeded means fast but non-reproducible. The whole point of the module is that the seeded path is a faithful clone of NumPy, so the higher-level estimators inherit scikit-learn parity for free.

How it works


    
  
  1. Seed the 624-word state (init). The constructor coerces the seed to unsigned 32-bit (seed >>> 0) and calls init. State word mt[0] = seed; each subsequent word is mt[i] = (1812433253 · (mt[i-1] ^ (mt[i-1] >>> 30)) + i) & 0xffffffff. The multiply is done in floating point then masked to 32 bits, and index is set to N = 624 so the first draw forces a twist. This is exactly NumPy / Knuth’s init_genrand.
    2. 
  
  2. Refill the state when exhausted (twist). When index ≥ 624, twist() regenerates all 624 words. For each i: combine the high bit of mt[i] with the low 31 bits of mt[(i+1) mod 624] into x; compute x >>> 1, XOR in MATRIX_A = 0x9908b0df when x is odd; then mt[i] = mt[(i+M) mod 624] ^ that, with M = 397. Reset index to 0.
    3. 
  
  3. Temper and return one uint32 (next_uint32). Read y = mt[index], increment index, then apply four tempering operations: y ^= y >>> 11; y ^= (y << 7) & 0x9d2c5680; y ^= (y << 15) & 0xefc60000; y ^= y >>> 18. Return y >>> 0 to keep it unsigned. These masks are identical to NumPy’s.
    4. 
  
  4. Build a 53-bit float in [0,1) (next_float). Draw two uint32s. a = first >>> 5 (top 27 bits), b = second >>> 6 (top 26 bits). Return (a · 67108864 + b) · 2-53 (the literal 1.1102230246251565e-16, with 67108864 = 226). This reproduces NumPy’s random_sample exactly — two words per float.
    5. 
  
  5. Bounded integer with rejection sampling (next_int). Validate that max is a positive 32-bit integer. Compute threshold = (232 − bound) mod bound. Loop: draw r = next_uint32(); if r ≥ threshold return r mod bound, else retry. This discards the low-end values that would cause modulo bias, matching NumPy’s bounded-integer routine exactly.
    6. 
  
  6. Expose a RandomStream (make_random_stream). If the seed is undefined, return { rand: Math.random, rand_int: floor(Math.random()*max) } — fast and non-deterministic. If a seed is given, construct MT19937(seed >>> 0) and wrap it: rand → next_float(), rand_int → next_int(max). All consumers depend only on those two methods.
    7.

The math



Seeding recurrence (init_genrand)


mt[i] ← (1812433253 · (mt[i−1] ⊕ (mt[i−1] >> 30)) + i) ∧ 0xffffffff,   i = 1 … 623


A linear-congruential-style fill of the 624-word state from one seed, with mt[0] = seed. The term mt[i−1] >> 30 shifts the top two bits down so they participate in the multiply; the XOR (⊕) mixes those high bits into the low end before multiplying by the constant 1812433253, so even seeds differing by one bit diverge quickly. The ∧ 0xffffffff truncates the product back to 32 bits (the multiply itself is computed in floating point first to avoid JS’s signed 32-bit overflow).



Twist (state recurrence)


x ← (mt[i] ∧ 0x80000000) ∨ (mt[(i+1) mod N] ∧ 0x7fffffff)
mt[i] ← mt[(i+M) mod N] ⊕ (x >> 1) ⊕ (x ∧ 1 ? 0x9908b0df : 0)


The core generalized-feedback-shift-register step, with N = 624 and M = 397. UPPER_MASK = 0x80000000 selects just the top bit of word i; LOWER_MASK = 0x7fffffff selects the low 31 bits of word i+1; the ∨ concatenates them into a 32-bit x. We then shift x right by one, conditionally XOR the twist matrix constant MATRIX_A = 0x9908b0df when x is odd, and fold in the “middle” word mt[i+397]. Indices wrap modulo 624, which is why the sweep visibly loops around near indices 624 and 397.



Tempering


y ← y ⊕ (y >> 11)
y ← y ⊕ ((y << 7) ∧ 0x9d2c5680)
y ← y ⊕ ((y << 15) ∧ 0xefc60000)
y ← y ⊕ (y >> 18)


An invertible bit-scrambling applied to each emitted word so successive outputs are equidistributed (uniform across all bit positions). >> and << are bit shifts; the ∧ with masks 0x9d2c5680 and 0xefc60000 gates which bits the leftward shifts are allowed to flip. The four shift amounts 11/7/15/18 and these two masks are the canonical MT19937 constants — identical to NumPy’s — which is exactly why the output stream lines up bit-for-bit.



53-bit uniform float


u ← (a · 226 + b) · 2-53,   a = w₁ >> 5 (27 bits),   b = w₂ >> 6 (26 bits)


Two 32-bit words w₁, w₂ are reduced to their top 27 and top 26 bits respectively (dropping noisy low bits), then packed into a 53-bit mantissa — the full precision of an IEEE-754 double. Multiplying by 2-53 = 1.1102230246251565e-16 scales it into [0,1). This is exactly NumPy’s random_sample, and it means every float costs two uint32 draws.



Unbiased bounded integer


threshold ← (232 − bound) mod bound;   draw r until r ≥ threshold;   return r mod bound


A plain r mod bound over-represents the first (232 mod bound) values, because 232 rarely divides evenly into bound equal blocks. Rejection sampling fixes this: it discards the leftover low region [0, threshold) so only whole blocks remain, every output is equally likely, and the integer sequence matches NumPy exactly rather than just approximately.



k-means++ distance-proportional sampling


P(point i) ∝ D(i)2,  D(i)2 = minc ‖xᵢ − μₐ‖2;   pick i where r · Σⱼ D(j)2 ≤ cumᵢ,  r ~ U[0,1)


Each candidate center is drawn with probability proportional to its squared distance D(i)2 from the nearest already-chosen center. The seeded rand() produces r; multiplying by the total potential Σ D(j)2 throws a dart, and a binary search over the cumulative-distance array finds which point it lands in. Reproducing NumPy’s float stream is precisely what makes the chosen centroids — and therefore the final labels — match scikit-learn.

Edge cases & scikit-learn parity


    
  
  • Exhaustion / first draw: the constructor sets index = N, so the very first next_uint32 triggers a twist before any output — matching NumPy, where the first refill happens on first use.
    • 
  
  • Unsigned coercion everywhere: every return and state write applies >>> 0 because JS bitwise ops produce signed 32-bit ints; the seeding multiply 1812433253 * prev is done in float then masked with & 0xffffffff to avoid 32-bit overflow truncation.
    • 
  
  • next_int validation: throws "max must be a 32-bit positive integer" for a non-integer, max ≤ 0, or max > 0xffffffff. The rejection loop can in principle retry, but threshold is small so termination is fast.
    • 
  
  • Two words per float: next_float consumes exactly two uint32 draws (27-bit + 26-bit). Any consumer mixing rand() and rand_int() must call them in the same order as scikit-learn or the streams diverge.
    • 
  
  • Unseeded mode is NOT reproducible: make_random_stream() with no seed delegates to Math.random and gives no parity guarantees; only the seeded path matches scikit-learn.
    • 
  
  • k-means++ degenerate potential: when current_pot === 0 (all remaining points coincide with chosen centers) the code deterministically takes the first unused index instead of sampling, avoiding division by zero / NaN probabilities (kmeans.ts:185–195).
    • 
  
  • n_init seed offsetting: KMeans adds seed_offset to the base seed per restart; if base_seed is undefined it stays undefined (each restart unseeded), preserving the seeded-vs-unseeded distinction across all restarts.
    • 




The parity contract. This module exists specifically to clone NumPy’s legacy RandomState (the MT19937 backend that scikit-learn’s check_random_state ultimately uses). Parity is achieved by matching four things exactly: (1) the init_genrand seeding recurrence with constant 1812433253 and the >>> 30 XOR-shift; (2) the twist with N=624, M=397, MATRIX_A=0x9908b0df and the upper/lower bit masks; (3) the four tempering shifts/masks (0x9d2c5680, 0xefc60000, shifts 11/7/15/18) identical to NumPy; (4) the 53-bit float construction (top 27 + top 26 bits, scaled by 2-53) matching random_sample, plus rejection-sampled bounded integers to remove modulo bias the way NumPy does. With these in place, k-means++ chooses the same initial centroids as scikit-learn for a given random_state (the rand()*current_pot weighted draw and the local_trials = 2 + floor(log K) greedy candidate scheme mirror sklearn’s _kmeans_plusplus), so final labels and inertia match. SOM and Lanczos similarly become reproducible. The deliberate seam: an undefined seed falls back to Math.random, which sacrifices parity for speed when determinism is not requested.
