Spectral Clustering

Cluster points by their connectivity in a similarity graph: build an affinity graph, find the smoothest functions on it (the smallest eigenvectors of the normalized Laplacian), and run k-means on that low-dimensional embedding.

Intuition

Picture the data points as cities and the affinity matrix A as a road network, where the weight A[i,j] says how easy it is to travel between city i and city j. Spectral clustering does not look at where the cities sit in raw feature space (the way k-means does); it looks at how they are connected. Two points can be far apart in Euclidean distance yet belong to the same cluster if a dense chain of intermediate points connects them. That is why spectral clustering separates non-convex shapes — nested rings, two moons — that plain k-means fails on.

The mathematical trick is the graph Laplacian. Think of each cluster as a region of the graph that is densely connected inside but only weakly connected to the rest. The smoothest non-trivial functions on the graph — vectors that change very little between strongly connected neighbors — are nearly constant inside each cluster and jump between clusters. Those smooth functions are exactly the eigenvectors of the normalized Laplacian with the smallest eigenvalues. Stacking the k smallest eigenvectors as columns gives every point a k-dimensional coordinate (the embedding) in which the tangled clusters become compact, roughly spherical blobs.

Once you are in that embedding space the hard geometric problem has been transformed into an easy one, so a final ordinary k-means on the embedding rows recovers the labels. In an ideal block-structured graph with k disconnected components, the smallest k eigenvalues are exactly zero and their eigenvectors are piecewise-constant indicator vectors — so k-means recovers the components perfectly. Real graphs are noisy approximations of that ideal, which is precisely why the method works.

This implementation mirrors scikit-learn's SpectralClustering closely, including a subtle detail: after extracting eigenvectors of the symmetric normalized Laplacian it divides each row by √d_i to recover the random-walk-normalized embedding, and it does not L2-row-normalize before k-means. Row normalization is reserved for the discretize assignment, which is not the default here.

How it works

The whole pipeline lives in SpectralClustering.fit() (src/clustering/spectral.ts). It runs in a fixed sequence:

  1. Ingest and guard input. X is accepted as a tf.Tensor2D or number[][] and cast to float32. Reject n_clusters > n_samples. For non-kNN affinities, reject n_samples > max_samples (default 10000) because the dense affinity needs O(n²) memory; the sparse nearest_neighbors path skips this guard.
  2. Build the affinity graph A. Branch on params.affinity. nearest_neighbors takes the sparse CSR path compute_sparse_knn_affinity. Everything else goes through compute_affinity_matrix: rbf → Gaussian kernel; precomputed → validate and use X directly; a callable → call and validate. The default kNN k = max(1, round(log2(n))).
  3. Sanity-check affinity. Sum all entries; if the total is exactly 0 the graph has no edges, so throw. Callable / precomputed matrices are additionally checked to be square, symmetric within 1e-6, and non-negative.
  4. Detect connected components. detect_connected_components (dense) or detect_sparse_connected_components (sparse) returns num_components, is_fully_connected, and per-node component_labels. A disconnected graph emits a console.warn.
  5. Connected-components shortcut. If the graph is disconnected and num_components ≥ n_clusters, skip the eigensolver entirely: create_component_indicators turns the component labels into orthonormal piecewise-constant indicator columns and uses them directly as the embedding. This is the exact analytic solution for the zero-eigenvalue eigenspace.
  6. Build the normalized Laplacian. Otherwise, the dense path runs normalised_laplacian(A, true): it zeros A's diagonal, computes degrees, forms L = I − D⁻¹ᐟ² A D⁻¹ᐟ², and returns sqrt_degrees = D⁻¹ᐟ². The sparse path builds a matrix-free operator whose matvec applies L without ever materializing it.
  7. Extract smallest eigenvectors. Request num_eigenvectors = max(n_clusters, num_components). A matrix-free operator always uses Lanczos; a dense matrix uses Lanczos when n > 100 and k < n/3, otherwise the dense Jacobi path. Lanczos over-fetches k+5 pairs to capture near-zero eigenvalues, with a Jacobi fallback for dense inputs.
  8. Degree-rescale the embedding. Slice the first n_clusters eigenvector columns, then divide each row by D¹ᐟ² = pow(sqrt_degrees, -1). This converts the symmetric-Laplacian eigenvectors into the random-walk embedding, matching sklearn's spectral_embedding recovery step. No unit-length row normalization is applied.
  9. Final k-means. Run KMeans on the embedding rows directly, with n_init defaulting to 10 and the supplied random_state. labels_ stores the result. Optional alternative paths exist: use_validation (with n_clusters ≥ 3) sweeps k-means seeds keeping the best validation score; intensive_parameter_sweep (rbf only) sweeps gamma × metric × attempts.

The math

RBF affinity

A[i,j] = exp(−γ · ‖xi − xj‖²),   diag(A) = 1

A Gaussian kernel similarity. γ (gamma) scales how fast similarity decays with distance: large γ means only very near points stay connected. The default γ = 1 / n_features matches sklearn. The matrix is symmetrized as (A + Aᵀ)/2 and the diagonal is forced to exactly 1 (compute_rbf_affinity, src/graph/affinity.ts).

Symmetrized kNN affinity

A = 0.5 · (W + Wᵀ),   W[i,j] = 1 if j ∈ kNN(i) else 0

The directed k-nearest-neighbor graph W made symmetric. A mutual edge (both directions present) gets weight 1.0; a one-directional edge gets 0.5. Self-loops are included so isolated points stay connected. k is the dial: larger k fuses more of the graph together.

Degree

di = Σj A[i,j]  (the diagonal of A is zeroed first)

d_i is the total edge weight touching node i. The diagonal matrix D = diag(d) is the degree matrix. The code zeros A's diagonal before summing, matching scipy's csgraph_laplacian (laplacian.ts lines 53–59).

Symmetric normalized Laplacian

Lsym = I − D−1/2 · A · D−1/2

The matrix whose smallest eigenvectors give the smooth graph functions. D⁻¹ᐟ² divides each row and column by √d_i so high- and low-degree nodes are comparable. Isolated nodes (d_i = 0) use inverse-sqrt = 1 to avoid division by zero (laplacian.ts lines 62–66). The diagonal of L ends up exactly 1 for non-isolated nodes.

Eigenproblem

Lsym · uℓ = λℓ · uℓ,   0 = λ1 ≤ λ2 ≤ … ,   pick ℓ = 1…k

Take the k eigenvectors with the smallest eigenvalues. λ_ℓ measures how non-smooth eigenvector u_ℓ is across edges, so small λ means smooth. For a graph with c connected components, exactly c eigenvalues equal 0 and their eigenvectors span the component-indicator space. The size of the gap after λ_k (the eigengap) suggests the natural number of clusters.

Embedding recovery (degree rescale)

U[i, :] ← u(i) / √di    i.e. divide row i by D1/2

This converts the L_sym eigenvectors into the random-walk normalized embedding. In code it is U_selected.div(pow(sqrt_degrees, -1)) (spectral.ts lines 367–369): sqrt_degrees stores D⁻¹ᐟ², so pow(·, −1) recovers D¹ᐟ², and dividing the row by D¹ᐟ² is the same as multiplying by √d_i⁻¹… that is, dividing each coordinate by √d_i. This mirrors sklearn's spectral_embedding.

Final assignment

labels = KMeans(U, k),   minimize Σi ‖U[i,:] − clabel(i)‖²

Ordinary k-means on the embedding rows. Each point is now a k-dimensional vector that sits near one of k well-separated centroids, so k-means trivially recovers the original tangled clusters. No L2-row-normalization in the default k-means path.

The core slice-and-rescale that turns eigenvectors into the embedding (spectral.ts:361-370):

// sqrt_degrees stores D^(-1/2); pow(-1) recovers D^(1/2)
U = tf.tidy(() => {
  const U_selected = tf.slice(U_full, [0, 0], [-1, this.params.n_clusters]);
  const sqrt_deg     = tf.pow(sqrt_degrees_tensor, -1);     // = D^(1/2)
  const sqrt_deg_col = sqrt_deg.reshape([-1, 1]);
  return U_selected.div(sqrt_deg_col);             // divide each row by sqrt(d_i)
});

Edge cases & scikit-learn parity

Edge cases

All-zero affinityThrows 'Affinity matrix contains only zeros' — the graph has no edges (spectral.ts:210-214).
Disconnected, components ≥ kSkips eigensolving entirely and uses analytic component indicators; this is exact, not an approximation. If num_components < n_clusters it still warns but proceeds through the Laplacian path.
Over-fetch eigenvectorsRequests max(n_clusters, num_components) eigenvectors, then slices the first n_clusters columns — over-fetching keeps the near-zero eigenspace intact when components exist.
Isolated nodesDegree-0 nodes get inverse-sqrt-degree set to 1 in both dense and sparse Laplacian builders to avoid division by zero (laplacian.ts:62-66, 116-118).
Solver selectionLanczos only when n > 100 and k < n/3 for dense matrices; otherwise Jacobi. A matrix-free operator (sparse path) always uses Lanczos with no Jacobi fallback.
Zero-eigenvalue toleranceLanczos uses 1e-2 while the dense Jacobi path uses 1e-7 — different solvers count near-zero eigenvalues differently.
No row normalizationThe default k-means path does not L2-row-normalize the embedding; row normalization is reserved for discretize, which this implementation does not use by default.
Parameter mismatchesgamma with non-rbf affinity, n_neighbors with non-kNN affinity, and gamma/n_neighbors with precomputed all throw at construction.
RBF cleanupThe RBF diagonal is forced to exactly 1 and the matrix is symmetrized via (A+Aᵀ)/2 to kill numerical asymmetry.

scikit-learn parity

  • Default gamma = 1/n_features matches sklearn's RBF kernel inside SpectralClustering.
  • Default kNN neighbors = max(1, round(log2(n))) matches sklearn.
  • The normalized Laplacian zeros the input adjacency diagonal before normalization, matching scipy's csgraph_laplacian.
  • The embedding recovery divides eigenvectors by D¹ᐟ², mirroring sklearn's spectral_embedding rescaling of symmetric-Laplacian eigenvectors back to the random-walk solution.
  • Default n_init = 10 for the inner k-means matches sklearn's default.
  • No row L2-normalization before k-means, matching sklearn's assign_labels='kmeans' default (sklearn only row-normalizes for 'discretize').
  • Deterministic eigenpair processing (sign-fixing and ordering) in the Jacobi path keeps eigenvector signs stable for reproducible parity.
  • Fixed Lanczos random_seed = 42 makes the iterative path reproducible across runs.
