Pairwise Distance Matrix

Computes the full (n,n) distance matrix between every pair of input rows — using a vectorized Gram-matrix identity for Euclidean, broadcasted reductions for Manhattan and cosine, all inside memory-managed TensorFlow.js tidy scopes.

Intuition

You have n data points, each a row in a matrix of d features. Agglomerative linkage, spectral affinity, and k-medoids all need to know how far apart every pair of points is — an (n,n) symmetric table where entry D[i,j] is the distance from point i to point j.

The naive way builds an (n,n,d) cube — broadcast the points into every pairing, then reduce over the feature axis. That cube is enormous: for n=10,000 points in d=100 dimensions it is 10 billion floats (40 GB in float32). The whole trick is to never build that cube for the common Euclidean case.

The key insight is algebraic. The squared Euclidean distance ‖x − y‖² expands to ‖x‖² + ‖y‖² − 2·xᵀy. The cross term xᵀy for all pairs is exactly the Gram matrix G = X·Xᵀ — a single (n,n) matrix multiply that hardware loves. The norm terms ‖x‖² are just a length-n vector. So the entire squared-distance matrix is one matmul plus a row-vector plus a column-vector, broadcast and subtracted. No (n,n,d) cube ever exists.

There is a catch to internalize: that identity is exact in real arithmetic but not in float32. When two points are identical or very close, ‖x‖² + ‖y‖² − 2·xᵀy can come out as a tiny negative number from rounding, and the square root of a negative is NaN. So the code clamps the squared matrix to a floor of zero before taking the root. It then re-symmetrizes (averages D with Dᵀ) and zeroes the diagonal, because round-off can leave both imperfect — and downstream code assumes a clean symmetric zero-diagonal matrix.

For Manhattan and cosine there is no Gram shortcut, so the code falls back to the honest broadcast: expand to (n,1,d) and (1,n,d), let broadcasting form the implicit (n,n,d) comparison, and reduce over d. Those paths get the same symmetrize-and-zero-diagonal cleanup, so all three metrics return matrices of identical structure.

How it works

  1. Dispatch on metric. pairwise_distance_matrix switches on the metric string ('euclidean' default, 'manhattan', 'cosine'); unknown metrics throw. Euclidean delegates to the specialized pairwise_euclidean_matrix; the other two are handled inline.
  2. Euclidean — squared norms. squared_norms = points.square().sum(1).reshape([-1,1]) — the per-row ‖xᵢ‖² as an (n,1) column vector.
  3. Euclidean — Gram matrix. gram = points.matMul(points.transpose()) — the (n,n) matrix of all pairwise dot products xᵢᵀxⱼ.
  4. Euclidean — apply the identity. distances_squared = squared_norms + squared_normsᵀ − 2·gram. Broadcasting adds the column vector and its row-vector transpose, then subtracting 2·gram completes ‖xᵢ − xⱼ‖².
  5. Euclidean — clamp to zero. tf.maximum(distances_squared, 0) with a float32 scalar zero removes the tiny negatives from round-off so the next sqrt never returns NaN.
  6. Euclidean — square root. clamped.sqrt() converts squared distances to actual ℓ2 distances.
  7. Manhattan / cosine — broadcast path. expandDims(1) → (n,1,d) and expandDims(0) → (1,n,d). Manhattan is a.sub(b).abs().sum(-1); cosine is 1 − (a·b)/(‖a‖‖b‖+1e-8). Broadcasting produces the (n,n) result over the last axis.
  8. Symmetrize. Every metric computes dist_sym = (dist + distᵀ)/2 to wipe out asymmetric float round-off, guaranteeing D[i,j] == D[j,i] exactly.
  9. Zero the diagonal. Build mask = ones(n,n) − eye(n) and multiply elementwise, forcing the self-distance diagonal to exactly 0 regardless of accumulated error.
  10. Memory management. Every path is wrapped in tf.tidy so all intermediates — gram, norms, masks, the broadcast temporaries — are disposed immediately, leaving only the returned (n,n) Tensor2D.

The math

Squared Euclidean identity

‖xᵢ − xⱼ‖² = ‖xᵢ‖² + ‖xⱼ‖² − 2·xᵢᵀxⱼ

Expanding the square of a difference splits a distance computation into norm terms that depend on a single point and a cross term xᵢᵀxⱼ that couples the pair. The cross term is the only thing that needs every pairing — and it is precisely a matrix multiply.

Matrix form

D² = s·1ᵀ + 1·sᵀ − 2·G,  where s = diag(X·Xᵀ), G = X·Xᵀ

s is the column of squared row-norms ‖xᵢ‖²; G is the Gram matrix. s·1ᵀ broadcasts s across every column (the ‖xᵢ‖² contribution), 1·sᵀ broadcasts it across every row (the ‖xⱼ‖² contribution). In the code, s·1ᵀ and 1·sᵀ are just squared_norms and squared_norms.transpose() added together by broadcast.

Squared row norm

sᵢ = ‖xᵢ‖² = Σₖ xᵢₖ²

Each sᵢ is the sum of squared features for row i — computed as points.square().sum(1) and reshaped to (n,1) so broadcasting forms both the row and column contributions from the same vector.

Numerical clamp

D²_clamped ← max(D², 0)

Float32 round-off can make D² slightly negative for near-identical points; clamping to 0 prevents NaN from sqrt. Note this guard only matters on the Euclidean path — Manhattan and cosine compute differences directly and never subtract a cross term.

Euclidean distance

Dᵢⱼ = √( max( ‖xᵢ‖² + ‖xⱼ‖² − 2·xᵢᵀxⱼ , 0 ) )

The final per-entry value, after clamping and square root.

Manhattan (ℓ1)

Dᵢⱼ = Σₖ |xᵢₖ − xⱼₖ|

Sum of absolute coordinate differences, via a.sub(b).abs().sum(-1) over the broadcast (n,n,d) tensor.

Cosine distance

Dᵢⱼ = 1 − (xᵢ·xⱼ) / (‖xᵢ‖·‖xⱼ‖ + ε),  ε = 1e-8

One minus cosine similarity. The ε in the denominator guards against division by zero for zero-vectors, so a zero row yields a finite distance ≈ 1 rather than NaN/Inf.

Symmetrization + diagonal

D ← ( (D + Dᵀ)/2 ) ⊙ (1 − I)

Average with the transpose to enforce exact symmetry, then multiply by (ones − identity) elementwise to zero the self-distance diagonal. Downstream symmetric-matrix consumers (linkage, spectral affinity) rely on this being bit-exact.

Try it · why we clamp

Slide two points toward each other. The raw identity computes D² = ‖x‖²+‖y‖²−2·xᵀy in float32; as the points coincide, that value dips just below zero (a rounding artifact). Without the clamp, sqrt of a negative is NaN and it poisons the matrix. Toggle enable clamp to snap the negative to a clean 0.

Try it · symmetrize & zero the diagonal

A raw D straight from matmul carries bit-level asymmetry (D[i,j] ≠ D[j,i]) and a nonzero diagonal. Step 1 averages with the transpose — every mirrored pair fuses to one value. Step 2 multiplies by (ones − I) — the diagonal snaps to black zeros. Hover any cell to read its value and its mirror.

Edge cases & scikit-learn parity




CaseBehavior



Near-identical / identical pointsThe identity ‖x‖²+‖y‖²−2xᵀy can be a tiny negative float32; tf.maximum(·,0) clamps it so sqrt never yields NaN. Only the Euclidean path needs this — Manhattan/cosine compute differences directly.


Self-distanceThe diagonal is explicitly forced to exactly 0 via the (ones − eye) mask, not left to whatever round-off the formula produced.


Asymmetry from round-offMatmul and reductions can yield D[i,j] ≠ D[j,i] at the bit level; (D + Dᵀ)/2 restores exact symmetry, which downstream symmetric-matrix consumers (linkage, spectral affinity) rely on.


Cosine zero-vectorsA 1e-8 epsilon is added to the denominator ‖a‖·‖b‖ so a zero row yields a finite distance ≈ 1 rather than NaN/Inf.


Memory blow-upEuclidean avoids the (n,n,d) intermediate, but Manhattan and cosine still materialize it via expandDims broadcasting — markedly more memory-hungry for large n·d.


float32 precisionAll computation is float32 (the clamp scalar is explicitly typed 'float32'), so absolute distances carry single-precision error — the baseline for any sklearn comparison.


tidy scopingResults are returned out of tf.tidy, so the caller owns and must dispose the returned (n,n) tensor; all intermediates are auto-disposed.







scikit-learn parity. The Euclidean path mirrors sklearn.metrics.pairwise.euclidean_distances, which uses the identical ‖x‖²+‖y‖²−2xᵀy expansion and the same np.maximum(dist, 0) clamp on the squared distances before sqrt — the clamp here is a faithful reproduction of sklearn's numerical guard, not an ad-hoc fix. Cosine matches 1 − cosine_similarity; sklearn's cosine_distances adds no epsilon and relies on normalized vectors, so the 1e-8 epsilon is a small deliberate deviation for zero-vector robustness. Manhattan matches manhattan_distances (ℓ1). The explicit symmetrize and zero-diagonal steps go slightly beyond sklearn (which trusts its symmetric formulation) but produce results that are bit-stable and equal-or-cleaner — which matters because the repo's agglomerative and spectral code demands exact tie-breaking parity and a perfectly symmetric, zero-diagonal D.
