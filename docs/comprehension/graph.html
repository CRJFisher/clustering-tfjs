Graph & Laplacian Construction

Turn a point cloud into a weighted similarity graph, then encode that graph as a normalized Laplacian whose smallest eigenvectors reveal cluster structure.

Intuition

Spectral clustering does not cluster points directly — it clusters a graph built from the points. The first job of this module is to decide who is connected to whom, and how strongly. Two strategies are offered.

The RBF (Gaussian) kernel connects every pair with a weight that decays smoothly with squared distance, so nearby points are strongly linked and far points are nearly — but never exactly — zero. The k-nearest-neighbor (kNN) graph instead connects each point only to its k closest neighbors with weight 1, producing a sparse graph that respects local manifold structure. Either way the output is a symmetric affinity matrix A, where A[i,j] is the edge weight between sample i and j.

The kNN graph has a subtlety: “i is a neighbor of j” is not symmetric. If I pick my k nearest friends, one of them might not count me among theirs. This code resolves the asymmetry by averaging A with its transpose, A ← 0.5·(A + Aᵀ). The arithmetic falls out cleanly: each directed edge contributes 0.5, so a mutual edge sums to 1.0 while a one-sided edge stays at 0.5. This is the “inclusive” symmetrization sklearn uses for connectivity graphs — as opposed to “mutual” kNN, which would keep only edges present in both directions.

Once you have A, the Laplacian re-expresses the graph in a form whose spectrum (eigenvalues/eigenvectors) encodes connectivity. The degree of a node is the total weight attached to it; the degree matrix D is diagonal. The symmetric normalized Laplacian L = I − D⁻¹⁄² A D⁻¹⁄² is the workhorse. Its eigenvalues live in [0, 2], and the number of zero eigenvalues equals the number of connected components of the graph. The eigenvectors for those near-zero eigenvalues are (almost) constant within each component — exactly the indicator structure you want for clustering.

That last fact drives the connected-components machinery. If the graph is disconnected, the iterative eigensolver can produce noisy, near-degenerate eigenvectors for the zero block. So the code detects components explicitly via graph traversal (BFS) and, when the graph is split, synthesizes clean component-indicator vectors directly instead of trusting the solver. The sparse CSR representation exists so all of this can run on large graphs without ever materializing an n×n dense matrix.

How it works

  1. Choose an affinity mode. compute_affinity_matrix dispatches on the option: 'rbf' calls compute_rbf_affinity; 'nearest_neighbors' calls compute_knn_affinity with include_self=true.
  2. RBF path — squared distances to kernel. Compute pairwise Euclidean distances, square them, multiply by −gamma, and exponentiate: A = exp(−&gamma·‖x_i−x_j‖²). gamma defaults to 1/n_features.
  3. RBF path — enforce exact symmetry and unit diagonal. sym = 0.5·(A + Aᵀ) removes numerical asymmetry, then the diagonal is forced to exactly 1 via sym·(1−I) + I.
  4. kNN path — block-wise distance scan. compute_sparse_knn_affinity processes rows in blocks of BLOCK_SIZE=1024, computing squared distances with the identity ‖x−y‖² = ‖x‖²+‖y‖²−2·xᵀy to keep peak memory at O(b·n) instead of O(n²).
  5. kNN path — top-k selection with tie-breaking. For each row it negates squared distances and calls tf.topk (k if include_self else k+1). Returned indices are sorted ascending so equal-distance ties resolve toward the lower index, mirroring NumPy.
  6. kNN path — self handling and symmetrization. add_symmetrised_edge sets self-loops to weight 1; for i≠j it adds 0.5 to both (i,j) and (j,i). Mutual neighbors accumulate to 1.0, one-sided edges stay at 0.5. The result is assembled into CSR via sparse_matrix_from_row_maps.
  7. Build the degree vector. degree_vector computes deg[i] = Σ_j A[i,j] by summing along axis 1, after validating the matrix is square.
  8. Zero the diagonal before normalization. normalised_laplacian first masks out A's diagonal (A_no_diag = A·(1−I)), matching scipy's rule that input diagonal entries are ignored for normalization.
  9. Compute the symmetric normalized Laplacian. inv_sqrt = deg⁻¹⁄² with isolated nodes (deg=0) set to 1; scaling = inv_sqrt ⊗ inv_sqrt; L = I − A_no_diag·scaling. The diagonal of L is exactly 1 for non-isolated nodes.
  10. Detect connected components. detect_connected_components runs BFS over the affinity matrix, treating A[i,j] > tolerance (default 1e-2) as an edge, and labels each node with its component id.
  11. Substitute clean indicators when disconnected. If num_components > 1, create_component_indicators builds an n × min(num_components, n_clusters) matrix where node i in component c gets value 1/√size_c — instead of relying on the eigensolver for the degenerate zero block.
  12. Solve the eigenproblem. For connected graphs the smallest eigenvectors of L are extracted (smallest_eigenvectors; Jacobi for small dense, Lanczos for large). The sparse path uses sparse_normalised_laplacian_operator as a matrix-free operator for Lanczos.

The math

A[i,j] = exp(−γ · ‖xi − xj‖²)

RBF / Gaussian kernel. The edge weight decays with the squared Euclidean distance between samples i and j. γ (gamma) is the kernel width — large γ means a fast decay (only very close points connect), small γ means a slow decay (almost everything connects). It defaults to 1/n_features. Because the distance is 0 on the diagonal, exp(0)=1, so the diagonal is naturally 1 (and is forced to exactly 1 afterwards).

‖x − y‖² = ‖x‖² + ‖y‖² − 2 · xᵀy

Squared-distance identity. Rather than form the full n×n distance matrix, the kNN block scan computes squared distances for a block of rows via one matrix multiply (xᵀy) plus cached row norms (‖x‖², ‖y‖²). This avoids both the O(n²) materialization and the sqrt — squared distances preserve neighbor ordering, which is all tf.topk needs.

A ← 0.5 · (A + Aᵀ)

kNN symmetrization (inclusive). The directed kNN graph is symmetrized so each directed edge contributes 0.5. A mutual pair (both list each other) sums to 0.5 + 0.5 = 1.0; a one-sided pair keeps a single 0.5. Self-loops are set to 1. In code this is done incrementally inside add_symmetrised_edge rather than as a literal transpose.

deg[i] = Σj A[i,j],    D = diag(deg)

Degree. The degree of node i is the total edge weight incident on it — an axis-1 row sum. The diagonal degree matrix D stacks these on its diagonal.

L = I − D−1/2 · A · D−1/2

Symmetric normalized Laplacian. The core operator. D⁻¹⁄² A D⁻¹⁄² rescales every edge by 1/√(deg_i · deg_j), and subtracting from I places the result in [0,2]. The eigenvectors of the smallest eigenvalues carry the cluster structure. Crucially, A's diagonal is zeroed first, so L's diagonal is exactly 1 for every non-isolated node.

di−1/2 ← 1 if deg[i] = 0,   else deg[i]−1/2

Isolated-node guard. A node with no connections has degree 0, which would make D⁻¹⁄² divide by zero. Setting d⁻¹⁄² = 1 for those nodes leaves their row and column of L equal to the identity — a harmless isolated 1 on the diagonal.

(L v)[i] = v[i] − Σj≠i A[i,j] · di−1/2 · dj−1/2 · v[j]

Sparse matvec for L. The matrix-free product used by the Lanczos solver. It never forms L explicitly — it walks the CSR rows of A, skipping the diagonal term (col == row), and applies the D⁻¹⁄² scaling on the fly. This is what lets large graphs be solved without an n×n matrix in memory.

num_components = #{ λk ≈ 0 }

Component count via spectrum. The multiplicity of eigenvalue 0 of L equals the number of connected components. Here it is detected directly by BFS (cheaper and more robust than counting noisy near-zero eigenvalues).

fc[i] = 1/√|componentc|   if node i ∈ component c,   else 0

Component indicator value. When the graph splits, these clean normalized indicator vectors are substituted for the degenerate near-zero eigenvectors. Each node in component c gets 1/√(size_c) in column c; the 1/√size factor matches the unit-norm normalization the eigensolver would have produced.

Edge cases & scikit-learn parity

Edge cases

  • Isolated vertices (degree 0): inv_sqrt is set to 1 rather than dividing by zero, leaving that node's L row/column equal to the identity (laplacian.ts:62-66).
  • Diagonal of A is zeroed before normalization to match scipy/sklearn: input diagonal entries are ignored and replaced with zeros for normalization (laplacian.ts:53-57). This makes L's diagonal exactly 1 for connected nodes.
  • kNN tie-breaking: indices from tf.topk are sorted ascending so equal-distance neighbors resolve toward the lower index, mirroring NumPy/sklearn determinism (affinity.ts:160-168).
  • include_self=false: fetches k+1 neighbors then filters the self index, guaranteeing exactly k genuine neighbors even when self is the nearest point (affinity.ts:157, 176).
  • RBF affinity is never exactly sparse: far pairs are tiny but nonzero, so the diagonal is explicitly re-forced to 1 and symmetry re-imposed by averaging with the transpose to kill numerical drift (affinity.ts:42-46).
  • Disconnected graphs: rather than trusting the eigensolver on the degenerate zero eigenspace, components are found by BFS and replaced with clean 1/√size indicator vectors (component_indicators.ts), avoiding noisy embeddings.
  • Jacobi fast-path: a near-diagonal matrix (common for disconnected normalized Laplacians) skips all rotations and returns the diagonal directly (laplacian.ts:210-219); tiny negative eigenvalues from numerical noise are clamped to 0.
  • Input validation: k must be an integer in [1, n_samples); n_samples=0 throws; affinity/Laplacian inputs must be square (affinity.ts:84-98, laplacian.ts:20).
  • Sparse component detection uses an index-based queue walk (queue_idx, no shift) over CSR rows — same BFS semantics as the dense version but without dense materialization (connected_components.ts:69).
scikit-learn / scipy parity. Several choices mirror the reference exactly: (1) RBF gamma defaults to 1/n_features, the sklearn SpectralClustering default. (2) kNN graphs are symmetrized as 0.5·(A+Aᵀ) — sklearn's “inclusive” connectivity convention — giving mutual edges weight 1.0 and asymmetric edges 0.5. (3) The diagonal of A is zeroed before degree/normalization, matching scipy.sparse.csgraph.laplacian's documented behavior. (4) Isolated nodes get D⁻¹⁄²=1, the same graceful handling scipy uses. (5) kNN ties break toward the lower index to match NumPy argsort/argpartition ordering. (6) The symmetric form L = I − D⁻¹⁄²AD⁻¹⁄² matches sklearn's spectral_embedding(norm_laplacian=True). (7) Disconnected graphs substitute normalized component indicators, mimicking what sklearn's shift-invert ARPACK path yields for the zero eigenspace.
